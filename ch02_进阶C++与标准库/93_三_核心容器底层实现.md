## 三、核心容器底层实现

### 3.1 `std::vector`

- **底层**：连续动态数组（三个指针：`start`、`finish`、`end_of_storage`）
- **扩容策略**：GCC/libstdc++ 通常 2 倍增长，MSVC 通常 1.5 倍增长（实现相关）
- **`push_back` 均摊 O(1)**：因为扩容频率呈指数衰减
- **迭代器失效**：插入/删除非末尾元素时，该位置及之后的迭代器全部失效；扩容时所有迭代器失效
- **`shrink_to_fit`**：非绑定请求，实现可自由决定是否真正缩减容量

### 3.2 `std::deque`

- **底层**：分段连续数组（中控器 map + 多个固定大小的 buffer）
- **中控器**：指针数组，每个指针指向一块 buffer（通常 512 字节或一个元素大小对齐）
- **头尾插入 O(1)**：只需在对应 buffer 中操作，必要时分配新 buffer
- **随机访问 O(1)**：通过中控器索引计算
- **迭代器失效**：仅头尾插入/删除不使迭代器失效；中间操作使所有迭代器失效

### 3.3 `std::list` / `std::forward_list`

- **`list`**：双向链表，每个节点包含 `prev`、`next` 指针和数据
- **`forward_list`**：单向链表，每个节点只有 `next` 指针，无 `size()` 方法（O(n) 计算）
- **插入/删除 O(1)**（已知位置）：仅修改指针，不移动元素
- **迭代器稳定性**：插入/删除不使其他迭代器失效

### 3.4 `std::set` / `std::map`

- **底层**：红黑树（自平衡二叉搜索树）
- **插入/删除/查找 O(log n)**
- **有序遍历**：中序遍历得到有序序列
- **节点分配**：每个元素独立分配一个树节点（内存不连续，缓存不友好）
- **迭代器稳定性**：插入不使迭代器失效；删除仅使被删除元素的迭代器失效

### 3.5 `std::unordered_set` / `std::unordered_map`

- **底层**：哈希表（链地址法或开放寻址法，libstdc++ 用链地址法）
- **平均 O(1)** 插入/删除/查找，最坏 O(n)（哈希冲突严重时）
- **负载因子（load_factor）**：`size / bucket_count`，超过 `max_load_factor` 时 rehash
- **rehash**：重新分配桶数组并重新插入所有元素，所有迭代器失效
- **迭代器稳定性**：插入不使迭代器失效（除非触发 rehash）

### 3.6 `std::priority_queue`

- **底层**：`std::vector` + 堆算法（`std::push_heap` / `std::pop_heap`）
- **默认最大堆**：`top()` 返回最大元素
- **插入 O(log n)**：上浮（sift-up）调整
- **删除堆顶 O(log n)**：将末尾元素移到堆顶后下沉（sift-down）调整
- **自定义比较**：通过第三个模板参数指定，注意 `std::greater` 得到最小堆
