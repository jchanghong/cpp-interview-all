怎么使用这份 HTML 笔记

第一轮复习

先看每页顶部的“面试官为什么爱问”，理解题目背后的考察目标，不要只背结论。

第二轮复习

重点看表格、示意图和代码块，把容易混淆的概念拉开，例如拷贝与移动、值捕获与引用捕获、map 与 unordered_map。

面试前冲刺

只看每节的回答模板与风险点，练习在 30 到 60 秒内说清楚定义、场景和代价。

高频主题地图

1. 基础语法与对象模型

指针/引用、const/static、构造析构、虚函数、重载/重写/隐藏。这部分用于检验你是否真的理解 C++ 对象的生命周期和类型系统。

2. 现代 C++ 核心

RAII、move 语义、右值引用、智能指针、auto、decltype、constexpr、lambda。这里考查你是否停留在“会写 C 风格 C++”的阶段。

3. STL 与模板

容器底层结构、迭代器失效、复杂度、模板约束、SFINAE 与 concepts。这部分常被用来判断抽象能力与工程判断力。

4. 并发与底层

线程、互斥量、条件变量、原子、CAS、死锁、memory_order。偏中高级岗位中，这一部分区分度很高。

5. C++20 / C++23

Concepts、Ranges、Coroutines、Modules、jthread、span、expected。新标准通常不是一上来就问，但很容易在追问中出现。

6. 工程实践

异常安全、资源泄漏、性能优化、日志与调试、线上故障定位。真正的项目经验通常通过这部分体现出来。

7. 设计模式

单例、工厂、策略、观察者、适配器、装饰器。这里重点考查抽象设计能力，以及你是否知道“什么时候该用、什么时候别滥用”。
