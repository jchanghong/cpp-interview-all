## 数组是什么

数组是同一类型元素的有序集合。说人话就是：一排连在一起的、类型相同的变量。

```c
int arr[5] = {10, 20, 30, 40, 50};
```

这行代码做了什么？它在内存中开辟了一块连续的空间，存了5个 int 类型的值。

## 数组的内存布局

数组最大的特点就是**内存连续**。所有元素紧挨着排列，中间没有空隙。

```
一维数组内存布局：

int arr[5] = {10, 20, 30, 40, 50};

地址:   0x100  0x104  0x108  0x10C  0x110
       ┌──────┬──────┬──────┬──────┬──────┐
内容:  │  10  │  20  │  30  │  40  │  50  │
       └──────┴──────┴──────┴──────┴──────┘
        arr[0] arr[1] arr[2] arr[3] arr[4]

每个 int 占 4 字节，所以相邻元素的地址差 4
```

**为什么连续这么重要？** 因为连续内存可以利用 CPU 缓存（Cache）的局部性原理，访问速度更快。这也是为什么面试官会问"数组和链表的区别"——数组连续、链表不连续，各有优劣。

### 数组大小计算

```c
int arr[5] = {10, 20, 30, 40, 50};

// 计算数组元素个数（经典写法，必须掌握）
int len = sizeof(arr) / sizeof(arr[0]);  // 20 / 4 = 5
```

**注意：** 这个方法只在定义数组的那个作用域里有效！如果数组作为函数参数传进去，`sizeof` 就只能拿到指针的大小了（后面会讲）。

## 数组名与指针的关系

这是一个面试高频考点，也是很多初学者容易搞混的地方。

### 数组名"退化"为指针

在大多数表达式中，数组名会自动"退化"（decay）为指向首元素的指针。

```c
int arr[5] = {10, 20, 30, 40, 50};

printf("%p\n", arr);      // 数组首地址
printf("%p\n", &arr[0]);  // 首元素地址（和上面一样）
printf("%p\n", &arr);     // 整个数组的地址（值相同，但类型不同！）
```

`arr` 和 `&arr` 的值一样，但类型不同：
- `arr` 的类型是 `int *`（指向 int 的指针）
- `&arr` 的类型是 `int (*)[5]`（指向"含5个int的数组"的指针）

### 数组名和指针的三个关键区别

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

// 区别1：sizeof 不同
printf("%lu\n", sizeof(arr));  // 20（整个数组的大小）
printf("%lu\n", sizeof(p));    // 8（指针本身的大小，64位系统）

// 区别2：数组名是常量，不能被赋值
arr = p;    // 编译错误！
p = arr;    // 正确

// 区别3：对数组名取地址 &arr 和 &arr[0] 类型不同
// &arr 的类型是 int (*)[5]
// &arr[0] 的类型是 int *
```

### 用指针遍历数组

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;

for (int i = 0; i < 5; i++) {
    printf("*(p + %d) = %d\n", i, *(p + i));
}

// 也可以用指针自增
for (int *q = arr; q < arr + 5; q++) {
    printf("%d ", *q);
}
```

## 二维数组

二维数组就是"数组的数组"——每个元素本身也是一个数组。

### 内存布局

```c
int arr[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};
```

```
二维数组内存布局（行优先存储）：

地址:   0x100  0x104  0x108  0x10C  0x110  0x114
       ┌──────┬──────┬──────┬──────┬──────┬──────┐
内容:  │  1   │  2   │  3   │  4   │  5   │  6   │
       └──────┴──────┴──────┴──────┴──────┴──────┘
        [0][0] [0][1] [0][2] [1][0] [1][1] [1][2]
        ─────── 第一行 ─────── ─────── 第二行 ───────

二维数组在内存中也是连续的！先存第一行，再存第二行。
```

### 用指针访问二维数组

```c
int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};

// 方法1：下标访问
printf("%d\n", arr[1][2]);  // 6

// 方法2：指针访问
printf("%d\n", *(*(arr + 1) + 2));  // 6

// 拆解：
// arr       → 指向第一行（类型 int (*)[3]）
// arr + 1   → 指向第二行
// *(arr+1)  → 第二行首元素地址（类型 int *）
// *(arr+1)+2 → 第二行第三个元素地址
// *(*(arr+1)+2) → 第二行第三个元素的值
```

### 数组指针 vs 指针数组

这两个名字经常搞混，记住诀窍：看 `*` 和 `[]` 谁先结合。

```c
int *ptr_arr[5];    // 指针数组：5个元素的数组，每个元素是 int*
int (*arr_ptr)[5];  // 数组指针：指向"含5个int的数组"的指针
```

- `ptr_arr` 是数组，里面存的是指针
- `arr_ptr` 是指针，指向一个数组

## 数组作为函数参数

数组传给函数时，会退化为指针，**丢失长度信息**。

```c
// 下面三种写法完全等价
void func(int arr[], int len);
void func(int *arr, int len);
void func(int arr[5], int len);  // 5 会被编译器忽略！
```

所以传数组时**必须额外传长度**：

```c
void print_array(int *arr, int len) {
    for (int i = 0; i < len; i++) {
        printf("%d ", arr[i]);
    }
    printf("\n");
}

int main() {
    int arr[] = {1, 2, 3, 4, 5};
    print_array(arr, sizeof(arr) / sizeof(arr[0]));
    return 0;
}
```

## C语言字符串

C语言没有专门的字符串类型，字符串就是**以 `\0` 结尾的字符数组**。

```c
char str1[] = "hello";     // 自动在末尾加 '\0'，实际占6字节
char str2[] = {'h', 'e', 'l', 'l', 'o', '\0'};  // 手动加 '\0'
char *str3 = "hello";      // 指向字符串常量（只读！）
```

```
字符串内存布局：

char str[] = "hello";

地址:   0x100  0x101  0x102  0x103  0x104  0x105
       ┌──────┬──────┬──────┬──────┬──────┬──────┐
内容:  │ 'h'  │ 'e'  │ 'l'  │ 'l'  │ 'o'  │ '\0' │
       └──────┴──────┴──────┴──────┴──────┴──────┘
        str[0] str[1] str[2] str[3] str[4] str[5]

'\0' 是字符串结束标志，不计入字符串长度，但占一个字节
```

**注意区分：**

```c
char str1[] = "hello";   // 字符数组，内容可修改
str1[0] = 'H';           // 正确

char *str2 = "hello";    // 指向字符串常量，内容不可修改
str2[0] = 'H';           // 未定义行为！可能崩溃
```

## 常用字符串操作函数

这些函数都在 `<string.h>` 中，面试中经常手写实现。

### strlen — 求字符串长度

```c
size_t strlen(const char *s);
// 返回字符串长度，不包含 '\0'
```

手写实现：

```c
size_t my_strlen(const char *s) {
    size_t len = 0;
    while (s[len] != '\0') {
        len++;
    }
    return len;
}
```

### strcpy — 字符串拷贝

```c
char *strcpy(char *dest, const char *src);
// 把 src 拷贝到 dest，包括 '\0'
// 注意：dest 必须有足够的空间！
```

手写实现：

```c
char *my_strcpy(char *dest, const char *src) {
    char *ret = dest;  // 保存起始地址，用于返回
    while ((*dest++ = *src++) != '\0') {
        // 什么也不做，赋值在条件里完成了
    }
    return ret;
}
```

### strcmp — 字符串比较

```c
int strcmp(const char *s1, const char *s2);
// 逐个字符比较 ASCII 值
// 返回 0 表示相等，<0 表示 s1 < s2，>0 表示 s1 > s2
```

手写实现：

```c
int my_strcmp(const char *s1, const char *s2) {
    while (*s1 && (*s1 == *s2)) {
        s1++;
        s2++;
    }
    return *(unsigned char *)s1 - *(unsigned char *)s2;
}
```

### strcat — 字符串拼接

```c
char *strcat(char *dest, const char *src);
// 把 src 拼接到 dest 后面
// 注意：dest 必须有足够的空间！
```

### strncpy / strncat / strncmp — 安全版本

带 `n` 的版本可以指定最大操作长度，更安全：

```c
strncpy(dest, src, n);   // 最多拷贝 n 个字符
strncat(dest, src, n);   // 最多拼接 n 个字符
strncmp(s1, s2, n);      // 最多比较 n 个字符
```

## 常见面试题

### 面试题1：strlen 和 sizeof 的区别？

**标准答案：**

> `strlen` 是函数，在运行时计算字符串的实际长度（到 `\0` 为止，不包含 `\0`）。`sizeof` 是运算符，在编译时计算，返回变量或类型占用的字节数。
>
> ```c
> char str[] = "hello";
> printf("%lu\n", strlen(str));  // 5（不包含 '\0'）
> printf("%lu\n", sizeof(str));  // 6（包含 '\0'）
>
> char *p = "hello";
> printf("%lu\n", strlen(p));    // 5
> printf("%lu\n", sizeof(p));    // 8（指针大小，64位系统）
> ```

### 面试题2：strcpy 和 strncpy 的区别？strcpy 安全吗？

**标准答案：**

> `strcpy` 不会检查目标缓冲区大小，如果源字符串比目标缓冲区长，就会造成**缓冲区溢出**。`strncpy` 可以指定最大拷贝长度，更安全。
>
> 但 `strncpy` 也有坑：如果源字符串长度大于 n，它不会自动添加 `\0`。所以更推荐使用 `strncpy` 后手动确保字符串以 `\0` 结尾，或者使用 `snprintf` 替代。
>
> 在实际工程中，C++ 推荐使用 `std::string`，C语言推荐使用 `strlcpy`（如果平台支持）或自己封装安全版本。

### 面试题3：数组越界会怎样？

**标准答案：**

> 数组越界是**未定义行为**。可能的表现：
> 1. 程序崩溃（段错误 Segmentation Fault）
> 2. 读到脏数据，程序"看似正常"但结果错误
> 3. 悄悄修改了其他变量的值，导致难以排查的bug
>
> C/C++ 不会自动检查数组边界，这是性能和安全的权衡。所以写代码时要自己小心，或者使用工具（如 AddressSanitizer）来检测越界访问。

### 面试题4：实现一个反转字符串的函数

**标准答案：**

```c
void reverse_string(char *str) {
    if (str == NULL) return;
    int len = strlen(str);
    for (int i = 0; i < len / 2; i++) {
        char temp = str[i];
        str[i] = str[len - 1 - i];
        str[len - 1 - i] = temp;
    }
}
```

> 时间复杂度 O(n)，空间复杂度 O(1)。双指针从两端向中间交换。

## 踩坑提醒

### 坑1：字符串没有以 '\0' 结尾

```c
char str[5] = {'h', 'e', 'l', 'l', 'o'};  // 没有 '\0'！
printf("%s\n", str);  // 未定义行为，会一直打印直到遇到 '\0'
```

**正确做法：** 数组大小至少比字符串长度多1，或者直接用 `char str[] = "hello"`。

### 坑2：用 == 比较字符串

```c
char s1[] = "hello";
char s2[] = "hello";
if (s1 == s2) {  // 错误！比较的是地址，不是内容
    // ...
}
if (strcmp(s1, s2) == 0) {  // 正确
    // ...
}
```

### 坑3：strcpy 目标空间不够

```c
char dest[5];
char src[] = "hello world";
strcpy(dest, src);  // 缓冲区溢出！dest 只有5字节，src 有12字节
```

### 坑4：sizeof 在函数中失效

```c
void func(int arr[]) {
    printf("%lu\n", sizeof(arr));  // 输出 8（指针大小），不是数组大小！
}
```
