## 什么是指针

先别慌，指针其实一点都不难。让我用一个生活中的例子来解释。

想象一下，你住在一栋公寓楼里。你的房间号是"302"，这就是你的**地址**。而你本人，住在302房间里的那个人，就是**数据**。

指针，就是一张写着房间号的纸条。它本身不是你，但它记录了你在哪里。

在计算机里，内存就像一栋大楼，每个房间（字节）都有一个编号（地址）。**指针变量就是用来存储这些地址的变量。**

```
内存示意图：

地址:    0x1000  0x1001  0x1002  0x1003  0x1004  0x1005  0x1006  0x1007
        ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐
内容:   │  42  │      │      │      │0x1000│      │      │      │
        └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
          ↑                                ↑
        变量 a                           指针 p
        (int, 值为42)                   (int*, 存储a的地址)
```

看上面的图：
- 变量 `a` 存放在地址 `0x1000`，它的值是 `42`
- 指针 `p` 存放在地址 `0x1004`，它的值是 `0x1000`（也就是 `a` 的地址）

所以当我们说"p 指向 a"，意思就是 **p 里面存着 a 的地址**。

用代码来表示：

```c
int a = 42;       // 在内存中开辟一块空间，存放整数42
int *p = &a;      // &a 取出a的地址，赋值给指针p
```

这里有两个关键符号：
- `&` — **取地址符**，拿到变量在内存中的地址
- `*` — **解引用符**，通过地址找到那个地址里存的值

```c
printf("%d\n", *p);  // 输出 42，因为 *p 就是 "去p存储的地址那里，把值取出来"
```

## 指针的声明与初始化

### 基本声明

```c
int *p;          // 指向int类型的指针
char *s;         // 指向char类型的指针
double *d;       // 指向double类型的指针
void *vp;        // 无类型指针，可以指向任意类型
```

注意：`int *p` 中的 `*` 是跟着变量名的，不是跟着类型的。也就是说，`int* p, q` 声明的是一个指针 `p` 和一个普通变量 `q`，不是两个指针。建议写成 `int *p` 的格式，这样更清晰。

### 初始化

```c
int a = 10;
int *p1 = &a;        // 用变量的地址初始化，最常见的方式
int *p2 = NULL;      // 初始化为空指针，表示"不指向任何地方"
int *p3 = (int *)malloc(sizeof(int));  // 用动态分配的内存初始化
```

**非常重要：** 声明指针后一定要初始化！未初始化的指针（野指针）里面存着随机地址，访问它会导致未定义行为，程序可能崩溃，也可能"看似正常运行"但偷偷改了别的数据。

```c
int *p;           // 危险！p里面是随机值
*p = 42;          // 灾难！往随机地址写数据
```

### NULL 指针

`NULL` 是一个特殊的指针值，表示"不指向任何地方"。在C语言中，`NULL` 通常是 `(void *)0`。在C++中，推荐使用 `nullptr`。

```c
int *p = NULL;
if (p == NULL) {
    printf("p 是空指针\n");
}
```

## 指针的基本操作

### 取地址和解引用

```c
int a = 10;
int *p = &a;

printf("a 的值:     %d\n", a);       // 10
printf("a 的地址:   %p\n", &a);      // 0x7ffd...
printf("p 的值:     %p\n", p);       // 0x7ffd...（和&a一样）
printf("p 指向的值: %d\n", *p);      // 10（通过地址取值）
```

### 通过指针修改值

```c
int a = 10;
int *p = &a;
*p = 20;              // 通过指针修改a的值
printf("a = %d\n", a); // 输出 20
```

### 指针的运算

指针可以进行加减运算，但要注意：**指针加减的不是字节数，而是"元素个数"**。

```c
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;         // p 指向 arr[0]

printf("%d\n", *p);       // 10
printf("%d\n", *(p + 1)); // 20（p+1 不是地址+1字节，而是地址+sizeof(int)字节）
printf("%d\n", *(p + 2)); // 30
```

```
指针运算示意图：

arr:   [10]  [20]  [30]  [40]  [50]
        ↑
        p     p+1   p+2   p+3   p+4

地址:  0x100  0x104  0x108  0x10C  0x110
       (每个int占4字节，所以每次+4)
```

## 多级指针

指针也是变量，也要占内存，也有地址。所以可以有一个指针指向另一个指针，这就是**多级指针**。

### 二级指针

```c
int a = 42;
int *p = &a;       // 一级指针，指向a
int **pp = &p;     // 二级指针，指向p

printf("a  = %d\n", a);       // 42
printf("*p = %d\n", *p);      // 42
printf("**pp = %d\n", **pp);  // 42
```

```
多级指针示意图：

变量:    a        p          pp
值:     42     0x1000     0x2000
地址:  0x1000   0x2000     0x3000

pp ──→ p ──→ a
0x3000  0x2000  0x1000   值: 42

*pp  = p 的值 = 0x1000（a的地址）
**pp = *p 的值 = 42（a的值）
```

### 二级指针最常见的用途：在函数中修改指针

```c
// 错误示范：想在函数里分配内存，但传一级指针不行
void bad_alloc(int *p) {
    p = (int *)malloc(sizeof(int));  // 只修改了局部变量p，外面的指针没变！
}

// 正确做法：传二级指针
void good_alloc(int **pp) {
    *pp = (int *)malloc(sizeof(int));  // 通过二级指针修改外面的指针
    **pp = 42;
}

// 调用
int *p = NULL;
good_alloc(&p);       // 传p的地址
printf("%d\n", *p);   // 42
free(p);
```

### 三级指针（了解一下就好）

三级指针在实际开发中几乎用不到，面试中偶尔会问到。原理和二级指针一样，就是再多套一层。

```c
int a = 42;
int *p = &a;
int **pp = &p;
int ***ppp = &pp;

printf("%d\n", ***ppp);  // 42
```

## 函数指针

函数指针是指向函数的指针。没错，函数在内存中也有地址，也可以用指针指向它。

### 基本用法

```c
// 先定义一个函数
int add(int a, int b) {
    return a + b;
}

// 定义函数指针
int (*fp)(int, int) = add;   // fp 指向 add 函数

// 通过函数指针调用函数
int result = fp(3, 5);       // 等价于 add(3, 5)
printf("%d\n", result);      // 8
```

函数指针的声明语法看起来有点吓人，让我帮你拆解一下：

```
int (*fp)(int, int)
 │   │    │    │
 │   │    │    └── 参数类型列表
 │   │    └── 指针名
 │   └── 这是一个指针（括号不能省！）
 └── 返回值类型
```

**注意：** `int (*fp)(int, int)` 和 `int *fp(int, int)` 完全不同！后者是函数声明，返回值是 `int*`。

### 用 typedef 简化

```c
typedef int (*MathFunc)(int, int);  // 给函数指针类型起个别名

MathFunc fp1 = add;
MathFunc fp2 = NULL;

int result = fp1(3, 5);  // 8
```

### 函数指针数组

```c
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int mul(int a, int b) { return a * b; }
int div(int a, int b) { return b != 0 ? a / b : 0; }

// 函数指针数组
int (*ops[])(int, int) = {add, sub, mul, div};

// 通过下标调用
for (int i = 0; i < 4; i++) {
    printf("ops[%d](10, 3) = %d\n", i, ops[i](10, 3));
}
// 输出：
// ops[0](10, 3) = 13
// ops[1](10, 3) = 7
// ops[2](10, 3) = 30
// ops[3](10, 3) = 3
```

### 回调函数

函数指针最常见的用途就是实现**回调函数**——把一个函数作为参数传给另一个函数。

```c
// 通用的排序函数，最后一个参数是比较函数
void bubble_sort(int arr[], int n, int (*compare)(int, int)) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (compare(arr[j], arr[j + 1]) > 0) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

// 升序比较
int cmp_asc(int a, int b) { return a - b; }

// 降序比较
int cmp_desc(int a, int b) { return b - a; }

// 使用
int arr[] = {5, 2, 8, 1, 9};
bubble_sort(arr, 5, cmp_asc);   // 升序排列
bubble_sort(arr, 5, cmp_desc);  // 降序排列
```

## 指针与引用的区别

引用是 C++ 引入的概念，C语言没有引用。引用可以理解为"给变量起了一个别名"。

```cpp
int a = 10;
int &ref = a;   // ref 是 a 的引用（别名）
ref = 20;        // 等价于 a = 20
```

### 对比表格

| 特性 | 指针 | 引用 |
|------|------|------|
| 本质 | 存放内存地址的变量 | 变量的别名 |
| 是否必须初始化 | 可以不初始化（但强烈建议初始化） | 必须在声明时初始化 |
| 能否为空 | 可以为 NULL | 不能为空 |
| 能否重新指向其他变量 | 可以随时改变指向 | 一旦绑定不能更改 |
| 是否占用内存 | 占用内存（通常4或8字节） | 不一定占用额外内存（编译器优化） |
| 是否需要解引用 | 需要用 `*` 解引用 | 直接使用，像普通变量一样 |
| 支持多级 | 支持多级指针（`**p`） | 没有多级引用 |
| sizeof 结果 | 指针本身的大小（4或8字节） | 所绑定变量的大小 |
| 自增自减 | `p++` 指向下一个地址 | `ref++` 是变量值自增 |
| 安全性 | 较低（可能出现野指针、空指针） | 较高（不存在空引用） |
| 适用场景 | 需要动态分配、数组操作、可能为空 | 函数参数传递、不需要改变指向 |

### 代码对比

```cpp
// 指针版本
void swap_pointer(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
// 调用：swap_pointer(&x, &y);

// 引用版本（C++）
void swap_reference(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}
// 调用：swap_reference(x, y);  // 更简洁，不用取地址
```

### 面试中怎么回答

面试官问"指针和引用的区别"时，建议这样回答：

> 指针和引用有三个最核心的区别：
> 1. **引用必须在声明时初始化，指针可以不初始化。** 这意味着引用不可能为空，而指针可能为 NULL。
> 2. **引用一旦绑定就不能改变指向，指针可以随时改变。** 引用就是变量的别名，从头到尾都代表同一个变量。
> 3. **指针需要解引用才能访问值，引用直接使用。** 所以引用在语法上更直观。
>
> 在实际开发中，能用引用就用引用，更安全更简洁。但如果参数可能为空、或者需要改变指向，就必须用指针。

## 常见面试题

### 面试题1：指针和引用的区别？

**标准答案：**

> 见上面的详细对比。核心回答三点：引用必须初始化、引用不能改变指向、引用不需要解引用。补充说明使用场景：能用引用就用引用，需要空值或改变指向时用指针。

### 面试题2：什么是野指针？怎么避免？

**标准答案：**

> 野指针是指向了不可用内存的指针。常见的产生原因有三种：
> 1. 指针未初始化：声明了指针但没有赋值，里面是随机地址
> 2. 指针被 free/delete 后没有置空：内存释放了，但指针还指向那块地址
> 3. 指针指向的变量已经超出作用域：比如指向了函数内的局部变量，函数返回后局部变量销毁
>
> 避免方法：
> - 声明指针时立即初始化为 NULL
> - free/delete 后立即将指针置为 NULL
> - 不要返回指向局部变量的指针

### 面试题3：函数指针有什么用？

**标准答案：**

> 函数指针主要有三个用途：
> 1. **回调函数**：把函数作为参数传给另一个函数，比如 qsort 的比较函数、信号处理函数
> 2. **策略模式**：运行时根据条件选择不同的函数执行，比如计算器程序中根据运算符选择加减乘除
> 3. **接口解耦**：模块之间通过函数指针交互，降低耦合度
>
> 在C++中，函数指针很多时候被 `std::function` 和 lambda 表达式替代，但在C语言和底层开发中，函数指针仍然非常重要。

### 面试题4：const 指针和指针 const 有什么区别？

**标准答案：**

> 这个问题的诀窍是看 `const` 在 `*` 的左边还是右边：
> - `const int *p`（或 `int const *p`）：const 在 `*` 左边，表示指针指向的值不能通过指针修改，但指针可以指向别的地方。叫"指向常量的指针"。
> - `int * const p`：const 在 `*` 右边，表示指针本身不能改变指向，但可以通过指针修改值。叫"指针常量"。
> - `const int * const p`：两个都不能改，既不能改值也不能改指向。
>
> 记忆技巧：从右往左读。`const int *p` 读作 "p is a pointer to int const"，即 p 是一个指针，指向常量 int。

### 面试题5：数组名和指针有什么区别？

**标准答案：**

> 数组名在大多数情况下会"退化"为指向首元素的指针，但它们有两个关键区别：
> 1. **sizeof 不同**：`sizeof(数组名)` 得到整个数组的大小，`sizeof(指针)` 得到指针本身的大小（4或8字节）
> 2. **数组名是常量**：数组名不能被赋值，不能做左值，而指针可以改变指向
>
> ```c
> int arr[5] = {1, 2, 3, 4, 5};
> int *p = arr;
> printf("%lu\n", sizeof(arr));  // 20（5个int，每个4字节）
> printf("%lu\n", sizeof(p));    // 8（64位系统上指针8字节）
> arr = p;  // 编译错误！数组名不能被赋值
> p = arr;  // 正确
> ```

## 踩坑提醒

### 坑1：忘记初始化指针

```c
int *p;        // 野指针！里面是随机值
printf("%d", *p);  // 未定义行为，可能崩溃
```

**正确做法：** 声明时立即初始化为 NULL。

### 坑2：free 后继续使用

```c
int *p = (int *)malloc(sizeof(int));
*p = 42;
free(p);
printf("%d", *p);  // 悬垂指针！内存已释放
```

**正确做法：** free 后立即置 NULL。

```c
free(p);
p = NULL;
```

### 坑3：混淆指针类型

```c
int a = 0x12345678;
char *p = (char *)&a;
printf("%x\n", *p);  // 输出 78（只读了一个字节！）
```

不同类型的指针解引用时，读取的字节数不同。`int*` 读4字节，`char*` 读1字节。

### 坑4：返回局部变量的地址

```c
int *bad_func() {
    int local = 42;
    return &local;  // 危险！local 在函数返回后就被销毁了
}

int *good_func() {
    int *p = (int *)malloc(sizeof(int));
    *p = 42;
    return p;  // 正确，返回的是堆上的内存
}
```

### 坑5：指针加减越界

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;
p = p + 10;        // 越界！指向了数组外面的内存
printf("%d", *p);  // 未定义行为
```
