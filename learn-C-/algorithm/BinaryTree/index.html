1. 二叉树的基础概念

二叉树是每个节点最多只有两个孩子的树结构，这两个孩子通常称为左子树和右子树。 它最大的特点是天然适合递归定义： 一棵二叉树 = 根节点 + 左子树 + 右子树。

这句定义非常关键，因为它直接决定了我们的解题方式。很多题都能拆成： “先解决左子树，再解决右子树，最后把答案在根节点处汇总”。

理解二叉树最重要的不是记住术语，而是牢记： 每个节点都可以被当成一棵新的子树的根。
  • 节点深度：从根走到该节点经过的边数。
  • 节点高度：从该节点走到最深叶子经过的边数。
  • 叶子节点：没有左右孩子的节点。
  • 满二叉树：每一层都满。
  • 完全二叉树：除了最后一层外都满，最后一层从左向右连续。

深度与高度

深度看“从根往下走了多远”，高度看“从当前节点往下还能走多远”。

满二叉树

每一层都被填满，每个非叶子节点都有两个孩子。

完全二叉树

最后一层可以不满，但必须从左到右连续，不能中间缺位。

1.1 链式存储的标准定义

算法题中最常用的是链式结构，每个节点保存一个值以及左右孩子指针。 这是二叉树题目的“基本单位”，后面所有遍历、构建、搜索，都是围绕这个结构展开。 

struct TreeNode {
    int val;
    TreeNode* left;
    TreeNode* right;

    TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
};

1.2 二叉树的常见性质

性质 说明
第 i 层最多节点数 2^(i - 1)，根节点视为第 1 层。
高度为 h 的二叉树最多节点数 2^h - 1。
完全二叉树的数组存储 若下标从 1 开始，则左孩子是 2*i，右孩子是 2*i+1。
递归本质 利用函数调用栈，自动保存从根到当前节点的路径状态。

2. 二叉树遍历

遍历是二叉树最基础也最重要的部分。所有操作本质上都在回答一个问题： 我们以什么顺序访问每个节点？

2.1 深度优先遍历 DFS

深度优先遍历会沿着一条路径走到尽头，再回退回来继续搜索。由于二叉树只有左、右两个方向， DFS 的核心区别在于“根节点是在左子树前访问，还是在左右子树之间访问，还是在右子树后访问”。

  • 前序：根 → 左 → 右
  • 中序：左 → 根 → 右
  • 后序：左 → 右 → 根

DFS 的感觉

DFS 会先沿一条分支尽量往深处走，再回退处理其他分支。

BFS 的感觉

BFS 按“层”扩散，先访问当前层所有节点，再进入下一层。

2.2 三种递归写法

void preorder(TreeNode* root, vector<int>& ans) {
    if (!root) return;
    ans.push_back(root->val);
    preorder(root->left, ans);
    preorder(root->right, ans);
}

void inorder(TreeNode* root, vector<int>& ans) {
    if (!root) return;
    inorder(root->left, ans);
    ans.push_back(root->val);
    inorder(root->right, ans);
}

void postorder(TreeNode* root, vector<int>& ans) {
    if (!root) return;
    postorder(root->left, ans);
    postorder(root->right, ans);
    ans.push_back(root->val);
}
这三段代码唯一不同的地方，是 ans.push_back(root->val) 放置的位置。 因此你可以把“前中后序”理解成：根节点的处理时机不同。

2.3 迭代写法为什么需要栈

递归本质上依赖系统调用栈。当我们手动改成迭代时，需要自己维护“回来的路”，所以一般使用 stack。 这也是为什么二叉树的 DFS 和栈天然契合。 

vector<int> preorderIter(TreeNode* root) {
    vector<int> ans;
    if (!root) return ans;

    stack<TreeNode*> st;
    st.push(root);

    while (!st.empty()) {
        TreeNode* node = st.top();
        st.pop();
        ans.push_back(node->val);

        if (node->right) st.push(node->right);
        if (node->left) st.push(node->left);
    }
    return ans;
}

这里要先压右孩子，再压左孩子，因为栈是后进先出。这样弹出时才会先访问左子树，符合前序顺序。

2.4 层序遍历为什么使用队列

层序遍历是按层从上到下、从左到右访问节点。这个顺序天然符合“先进先出”，所以一般使用 queue。 

vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> ans;
    if (!root) return ans;

    queue<TreeNode*> q;
    q.push(root);

    while (!q.empty()) {
        int sz = q.size();
        vector<int> level;
        for (int i = 0; i < sz; ++i) {
            TreeNode* node = q.front();
            q.pop();
            level.push_back(node->val);

            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        ans.push_back(level);
    }

    return ans;
}

3. 二叉树题目的递归思想

二叉树递归通常分为两类思路，这也是很多人刷题时最容易混淆的地方。

3.1 自顶向下

从根节点携带状态往下传，例如当前深度、当前路径和、路径数组。 典型题目：路径总和、收集根到叶子的路径、判断某节点是否在某层。 

void dfs(TreeNode* root, int depth) {
    if (!root) return;
    // 使用当前层信息
    dfs(root->left, depth + 1);
    dfs(root->right, depth + 1);
}

3.2 自底向上

先递归求出左右子树的答案，再在当前节点合并结果。 典型题目：树高、平衡树、直径、最大路径和、最近公共祖先。 

int height(TreeNode* root) {
    if (!root) return 0;
    int lh = height(root->left);
    int rh = height(root->right);
    return max(lh, rh) + 1;
}
判断使用哪种递归方式，有一个简单经验： 如果答案依赖“祖先传下来的信息”，用自顶向下； 如果答案依赖“左右子树返回的结果”，用自底向上。

自顶向下

像把“深度、路径和、路径数组”从父节点传给孩子节点。

自底向上

像“高度、直径、是否平衡”这种答案，通常先由左右子树算出再合并。

3.3 递归题的统一思考模板

  1. 定义递归函数的返回值含义。例如：dfs(root) 返回以 root 为根的子树高度。
  2. 明确终止条件。通常空节点返回 0、false、nullptr 或其他边界值。
  3. 假设左子树和右子树已经解决，思考如何合并。
  4. 检查当前节点是否需要更新全局变量，例如最大值、路径答案。

4. 二叉搜索树 BST

二叉搜索树是最常考的特殊二叉树。其核心性质是： 左子树所有节点值小于根节点，右子树所有节点值大于根节点。 这让它具备了类似“二分查找”的能力。

4.1 为什么中序遍历是有序的

中序遍历顺序是“左、根、右”。而 BST 保证左边都小、右边都大， 所以中序遍历得到的序列一定是递增序列。这是检验 BST、恢复 BST、求第 k 小值时的基础。 

bool isValidBST(TreeNode* root, long long low, long long high) {
    if (!root) return true;
    if (root->val <= low || root->val >= high) return false;
    return isValidBST(root->left, low, root->val) &&
           isValidBST(root->right, root->val, high);
}

这里不能只比较当前节点与左右孩子，因为 BST 约束的是整棵子树，不只是相邻节点。 因此要传递上下界。

搜索

TreeNode* searchBST(TreeNode* root, int val) {
    while (root && root->val != val) {
        root = (val < root->val) ? root->left : root->right;
    }
    return root;
}

插入

TreeNode* insertIntoBST(TreeNode* root, int val) {
    if (!root) return new TreeNode(val);
    if (val < root->val) root->left = insertIntoBST(root->left, val);
    else root->right = insertIntoBST(root->right, val);
    return root;
}

求第 k 小

int kthSmallest(TreeNode* root, int k) {
    stack<TreeNode*> st;
    while (true) {
        while (root) {
            st.push(root);
            root = root->left;
        }
        root = st.top();
        st.pop();
        if (--k == 0) return root->val;
        root = root->right;
    }
}
BST 平均操作复杂度是 O(log n)，但如果退化成链表，最坏会变成 O(n)。 所以在工程中常会引入 AVL、红黑树等平衡策略。

BST 搜索方向

查找值比根大就去右子树，比根小就去左子树，路径像二分查找一样收缩。

5. 二叉树经典题型与实现

5.1 求树的最大深度

这是最经典的自底向上问题。当前树的高度，等于左右子树高度较大者再加一。 

int maxDepth(TreeNode* root) {
    if (!root) return 0;
    return max(maxDepth(root->left), maxDepth(root->right)) + 1;
}

5.2 判断是否为平衡二叉树

平衡二叉树要求每个节点的左右子树高度差不超过 1。 这个题如果每次都单独求高度，会重复计算很多次。更高效的做法是： 递归过程中同时返回高度，一旦发现不平衡就返回特殊值。

平衡树定义

不是只看根节点，要对每一个节点都满足高度差不超过 1。

int heightOrFail(TreeNode* root) {
    if (!root) return 0;
    int lh = heightOrFail(root->left);
    if (lh == -1) return -1;
    int rh = heightOrFail(root->right);
    if (rh == -1) return -1;
    if (abs(lh - rh) > 1) return -1;
    return max(lh, rh) + 1;
}

bool isBalanced(TreeNode* root) {
    return heightOrFail(root) != -1;
}

5.3 最近公共祖先 LCA

最近公共祖先问题的关键理解是： 如果 p 和 q 分别出现在当前节点的左右子树中，那么当前节点就是答案。 如果只出现在一边，就把那一边的结果向上返回。

LCA 定义

从 p、q 往上追溯，第一次重合到的祖先节点就是最近公共祖先。

TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;

    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);

    if (left && right) return root;
    return left ? left : right;
}

5.4 二叉树直径

直径指任意两个节点之间最长路径的边数。经过某个节点的最长路径， 等于左子树高度 + 右子树高度，因此可以在求高度时顺便更新全局答案。

直径定义

直径不一定经过根节点，它是整棵树里最长的一条节点路径。

class DiameterSolver {
public:
    int ans = 0;

    int depth(TreeNode* root) {
        if (!root) return 0;
        int left = depth(root->left);
        int right = depth(root->right);
        ans = max(ans, left + right);
        return max(left, right) + 1;
    }

    int diameterOfBinaryTree(TreeNode* root) {
        depth(root);
        return ans;
    }
};

5.5 路径总和

这类题一般是自顶向下递归，把从根到当前节点的累计值传下去。 到叶子节点时判断是否满足条件。

路径总和定义

通常要求的是一条从根出发到叶子结束的路径，其节点值加起来等于目标值。

bool hasPathSum(TreeNode* root, int targetSum) {
    if (!root) return false;
    if (!root->left && !root->right) return targetSum == root->val;
    return hasPathSum(root->left, targetSum - root->val) ||
           hasPathSum(root->right, targetSum - root->val);
}

5.6 翻转二叉树

这是最直观的树结构修改题：对每个节点交换左右孩子即可。 

TreeNode* invertTree(TreeNode* root) {
    if (!root) return nullptr;
    swap(root->left, root->right);
    invertTree(root->left);
    invertTree(root->right);
    return root;
}

5.7 判断两棵树是否相同 / 是否对称

bool isSameTree(TreeNode* p, TreeNode* q) {
    if (!p && !q) return true;
    if (!p || !q || p->val != q->val) return false;
    return isSameTree(p->left, q->left) &&
           isSameTree(p->right, q->right);
}

bool mirror(TreeNode* a, TreeNode* b) {
    if (!a && !b) return true;
    if (!a || !b || a->val != b->val) return false;
    return mirror(a->left, b->right) &&
           mirror(a->right, b->left);
}

bool isSymmetric(TreeNode* root) {
    if (!root) return true;
    return mirror(root->left, root->right);
}

6. 二叉树的构造、序列化与还原

6.1 由前序 + 中序构造二叉树

前序遍历的第一个节点一定是根节点；在中序遍历中，根节点左边是左子树，右边是右子树。 于是我们可以递归切分区间构造整棵树。

构造原理

前序先给你根，中序告诉你左右子树边界，递归切分即可。

class BuildTree {
public:
    unordered_map<int, int> pos;

    TreeNode* build(vector<int>& preorder, int pl, int pr,
                    vector<int>& inorder, int il, int ir) {
        if (pl > pr) return nullptr;
        int rootVal = preorder[pl];
        int k = pos[rootVal];
        int leftSize = k - il;

        TreeNode* root = new TreeNode(rootVal);
        root->left = build(preorder, pl + 1, pl + leftSize,
                          inorder, il, k - 1);
        root->right = build(preorder, pl + leftSize + 1, pr,
                           inorder, k + 1, ir);
        return root;
    }

    TreeNode* buildTree(vector<int>& preorder, vector<int>& inorder) {
        for (int i = 0; i < (int)inorder.size(); ++i) pos[inorder[i]] = i;
        return build(preorder, 0, (int)preorder.size() - 1,
                     inorder, 0, (int)inorder.size() - 1);
    }
};

6.2 序列化与反序列化

所谓序列化，就是把树变成字符串，反序列化就是把字符串还原成树。 常见方法是用前序遍历，并为 null 节点写入占位符，例如 #。

序列化原理

值序列本身不够，还必须记录空节点，才能唯一恢复整棵树。

class Codec {
public:
    void serializeDfs(TreeNode* root, string& out) {
        if (!root) {
            out += "#,";
            return;
        }
        out += to_string(root->val) + ",";
        serializeDfs(root->left, out);
        serializeDfs(root->right, out);
    }

    TreeNode* deserializeDfs(queue<string>& q) {
        string cur = q.front();
        q.pop();
        if (cur == "#") return nullptr;

        TreeNode* root = new TreeNode(stoi(cur));
        root->left = deserializeDfs(q);
        root->right = deserializeDfs(q);
        return root;
    }

    string serialize(TreeNode* root) {
        string out;
        serializeDfs(root, out);
        return out;
    }

    TreeNode* deserialize(const string& data) {
        queue<string> q;
        string token;
        for (char ch : data) {
            if (ch == ',') {
                q.push(token);
                token.clear();
            } else {
                token.push_back(ch);
            }
        }
        return deserializeDfs(q);
    }
};

ReturnType dfs(TreeNode* root) {

    int sz = q.size();

        TreeNode* node = q.front(); q.pop();

中序模板

stack<TreeNode*> st;
while (root || !st.empty()) {

        st.push(root);
        root = root->left;
    }
    root = st.top(); st.pop();
    // 处理 root

7.1 刷题时最常见的误区

  • 只比较父子节点大小来判断 BST，忽略了整棵子树范围约束。
  • 递归函数定义不清楚，不知道返回值到底表示什么。
  • 把“节点数”和“边数”混用，导致树高和直径差 1。
  • 在路径类题目中没有正确回溯，导致路径数组被污染。
  • 层序遍历中没有记录当前层大小，结果不同层数据混在一起。

7.2 复杂度如何分析

操作 时间复杂度 空间复杂度
一次完整 DFS 遍历 O(n) O(h)，来自递归栈
一次完整 BFS 遍历 O(n) O(w)，w 为某层最大宽度
BST 搜索 / 插入 / 删除 平均 O(log n)，最坏 O(n) O(h)
学二叉树真正高效的方法，不是把所有题目死记硬背，而是先分辨题型： 遍历、路径、构造、性质判断、搜索、公共祖先、区间合并。 一旦题型明确，模板几乎就固定了。

8. 一份适合背诵的总结

核心原则

  • 二叉树是递归定义的数据结构。
  • 前中后序的本质区别，是根节点处理时机不同。
  • DFS 常配合栈或递归，BFS 常配合队列。
  • BST 的核心是范围约束，中序遍历递增。
  • 很多树题都是“左子树答案 + 右子树答案 + 根节点合并”。

实战口诀

  • 求高度、平衡、直径：自底向上。
  • 求路径、深度、累计值：自顶向下。
  • 按层输出、最短步数：优先想到 BFS。
  • 涉及有序、小于大于关系：优先检查是不是 BST。
  • 不会做时，先定义 dfs(root) 的含义。
