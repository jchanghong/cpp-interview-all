## 四、静态分析工具

### 4.1 clang-tidy

C++ 静态分析工具，检查代码风格、bug 模式、现代 C++ 用法等。

```bash
# 基本使用
clang-tidy src/main.cpp -- -std=c++17 -Iinclude

# 常用检查项
clang-tidy src/*.cpp --checks='modernize-*,performance-*,bugprone-*' \
    -- -std=c++17

# 与 CMake 集成（生成 compile_commands.json）
cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON ..
run-clang-tidy -p build
```

### 4.2 clang-format

代码格式化工具，保持团队代码风格一致。

```bash
# 格式化文件
clang-format -i src/main.cpp

# 检查格式（不修改）
clang-format --dry-run --Werror src/main.cpp
```

常用 `.clang-format` 配置：
```yaml
BasedOnStyle: Google
IndentWidth: 4
ColumnLimit: 100
```

### 4.3 Cppcheck

独立于编译器的静态分析工具，擅长发现潜在 bug。

```bash
cppcheck --enable=all --inconclusive src/
```
