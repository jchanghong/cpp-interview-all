# cppinter — 中国 C++ 社招面试资料库

## 仓库性质

这是一个 **C++ 社招面试准备资料库**（中国招聘市场），**不是代码仓库**。所有内容被 GPT 合并整理为每个目录下的单一 `README.md`，原文件仅存标题（作为章节名），实际已删除。

## 目录结构

| 目录 | 内容 | 注意事项 |
|------|------|----------|
| `编程语言基础/` | C/C++ 基础语法/关键字/编译链接 | 含大量百度网盘课程链接（可能失效且有版权风险） |
| `进阶C++与标准库/` | C++11/14/17 特性、STL 容器/算法、OOP、智能指针 | 大量 API 搬运（array/vector/forward_list 的逐方法示例）；已后补现代 C++ 专题、STL 底层原理、工程工具链 |
| `操作系统与Linux/` | 进程/线程/内存/IO/Cache/调度 | 已后补网络编程实战（Reactor/Proactor/io_uring/零拷贝） |
| `计算机网络/` | TCP/IP/HTTP/Socket/拥塞控制 | ⚠️ 顶部 200+ 跨目录 markdown 链接全部断裂（已替换为纯文本标题） |
| `数据库与缓存/` | MySQL InnoDB/B+Tree/Redis 数据结构/缓存策略 | 知识错误已修复（InnoDB FULLTEXT、Redis quicklist/listpack 等） |
| `数据结构与算法/` | 链表/树/堆/红黑树/排序/跳表/BloomFilter | 知识错误已修复（满二叉树定义、堆调整方向） |
| `算法刷题题解/` | 刷题建议/题目分类表 | 代码在外部源，不是完整 C++ 题解 |
| `设计模式与项目实践/` | 单例/工厂/适配器/内存池/ThreadPool | ⚠️ 含大量"面试鸭"React+Node.js 项目文档（非 C++）；单例示例已补充风险注释 |

## 已知关键约束

1. **所有跨目录链接断裂** — 原始 `.md` 文件已合并删除，URL 编码的相对链接（`problems__*.md`、`docs__*.md` 等）全部无效。修改时只应写纯文本标题。
2. **非 C++ 内容混入** — "面试鸭"章节是 React+Node.js 全文，如果清理要谨慎确认。
3. **百度网盘链接** — `编程语言基础/README.md` 有大量链接，涉及版权风险。
4. **内容来自多个仓库合并** — 可能存在重复、冲突、分类不当（如栈/队列出现在操作系统目录）。
5. **STL 章节是 API 搬运** — 早期来自 cplusplus.com，深层原理在后补章节中。
6. **单例代码示例有隐患** — 懒汉式非线程安全、使用 `0` 而非 `nullptr`、缺拷贝禁用。已加警告注释。

## 修改规范

- 所有知识性修改必须核实（优先用 cppreference 而非 CSDN）
- 新增内容使用 `---` 分隔，保持原有 `#` / `###` 标题层级
- C++ 版本标注要精确（如 random_shuffle C++14 废弃 C++17 移除）
- 不要删除旧文件名式标题（如 `# problems__xxx.md`），它们是合并来源标识
- 对不安全代码示例（单例、过时 API）补充注释而非直接修改代码
