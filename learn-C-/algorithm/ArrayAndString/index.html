1. 总览：为什么数组和字符串是算法基础

数组和字符串本质上都是“顺序存储的序列”。一旦你把题目理解为“在一个连续序列上维护信息”，很多题目的套路就会自然浮现出来。

随机访问

下标是核心

数组和字符串都可以在 O(1) 时间访问第 i 个元素，所以很多算法以“位置”为中心来设计，比如前缀和、双指针、滑动窗口、KMP。

顺序扫描

一次遍历能做很多事

如果能在扫描过程中维护某些状态，例如窗口和、字符频次、栈内单调性、前缀信息，就能把原本需要多重循环的问题压缩成线性复杂度。

局部到整体

维护当前最重要的信息

算法题常常不是在枚举所有方案，而是在维护“当前最有价值的局部信息”，然后逐步推出全局答案。滑动窗口和单调栈都属于这种思想。

题目特征 优先联想到的方法 本质
连续子数组 / 子串 滑动窗口、前缀和 维护一个区间的信息
原地删除、原地去重、两端收缩 双指针 用两个位置描述过程
统计出现次数、判断是否存在、配对 哈希表 用空间换时间
求区间和、频繁做区间修改 前缀和、差分 预处理累计信息
第一个满足条件的位置 二分查找 利用单调性找边界
下一个更大 / 更小元素 单调栈 丢弃永远不可能再有用的信息
模式匹配 KMP 利用模式串自身的重复结构
判断模型时最有效的一组问题是：这是不是连续区间？是否在找边界？是否需要快速判断“出现过”？是否存在可复用的前缀信息？是否存在单调性？是否在重复比较同样的字符？

2. 数组基础：连续存储带来的能力与代价

数组的最大特点是 连续内存。这带来两个直接后果：

能力：随机访问快

因为元素地址可以通过“首地址 + 偏移量”直接算出来，所以访问 a[i] 的复杂度是 O(1)。

代价：中间插入删除慢

如果在中间插入一个元素，后面的元素都要向后搬移；删除时也类似，因此复杂度通常是 O(n)。

示意图：数组中间插入时，后续元素整体右移

2.1 数组题的常见观察角度

从“值”看

例如：是否有重复、是否需要排序、是否要找两数和。此时常见工具是哈希、排序、二分。

从“位置”看

例如：区间和、最短区间、最长区间、某个元素能影响多远。此时常见工具是前缀和、窗口、单调栈。

3. 双指针与滑动窗口

双指针的本质是：用两个位置来刻画一个过程，从而避免重复枚举。滑动窗口可以看成双指针在“连续区间”问题中的一个特化版本。

3.1 双指针原理

如果暴力做法中存在很多重复比较，就很可能能用双指针优化。最常见的三种形式是：

快慢指针

适合原地删除、去重、按条件保留元素。一个指针负责读，一个指针负责写。

相向双指针

适合回文判断、反转数组、排序数组中找两数和。两个指针从两端向中间逼近。

同向双指针

适合维护区间，典型就是滑动窗口。右指针扩展区间，左指针在必要时收缩区间。

示意图：快慢指针做原地删除

// 原地删除所有等于 val 的元素
int removeElement(vector<int>& nums, int val) {
    int slow = 0;
    for (int fast = 0; fast < (int)nums.size(); ++fast) {
        if (nums[fast] != val) {
            nums[slow] = nums[fast];
            ++slow;
        }
    }
    return slow;
}
这个写法的关键在于：我们不关心被删除元素原来在哪，只关心“保留下来的元素最终要紧密排在前面”。因此 slow 指针只在保留元素时前进即可。

3.2 相向双指针

// 反转数组
void reverseArray(vector<int>& nums) {
    int left = 0, right = (int)nums.size() - 1;
    while (left < right) {
        swap(nums[left], nums[right]);
        ++left;
        --right;
    }
}

反转数组之所以适合相向双指针，是因为第 i 个位置和第 n-1-i 个位置互为目标位置，不需要额外结构即可完成交换。

3.3 滑动窗口原理

滑动窗口用于解决“连续区间”问题。它的关键不是“窗口”两个字，而是：随着右端点移动，我们能否快速更新当前区间信息；如果条件不满足，左端点是否能单调前进。

示意图：滑动窗口的扩展与收缩

// 和 >= target 的最短连续子数组长度
int minSubArrayLen(int target, vector<int>& nums) {
    int left = 0;
    long long sum = 0;
    int ans = INT_MAX;

    for (int right = 0; right < (int)nums.size(); ++right) {
        sum += nums[right];

        while (sum >= target) {
            ans = min(ans, right - left + 1);
            sum -= nums[left];
            ++left;
        }
    }

    return ans == INT_MAX ? 0 : ans;
}
这个题能用滑动窗口的前提是：数组元素为正数。因为只要区间和已经足够大，左端点右移只会让和减小，窗口状态具有单调性。如果数组里有负数，这个单调性就消失了，窗口往往不再适用。

3.4 最长无重复字符子串

这是字符串窗口题的代表。核心思想是：窗口中不能有重复字符，所以我们要维护每个字符最近一次出现的位置。

int lengthOfLongestSubstring(const string& s) {
    vector<int> last(256, -1);
    int left = 0, ans = 0;

    for (int right = 0; right < (int)s.size(); ++right) {
        unsigned char c = s[right];
        if (last[c] >= left) {
            left = last[c] + 1;
        }
        last[c] = right;
        ans = max(ans, right - left + 1);
    }
    return ans;
}

这里 left 不会回退，只会单调右移，因此依然是线性复杂度。

4. 前缀和与差分：把重复区间操作变成常数时间

4.1 前缀和原理

如果一个题目要多次求区间和，暴力每次都重新加一遍会很慢。前缀和的思路是：先把“从开头到当前位置的累计和”全部算好，之后任意区间和都能通过两次相减得到。

示意图：区间和 = 两个前缀之差

vector<long long> buildPrefix(const vector<int>& nums) {
    int n = nums.size();
    vector<long long> pre(n + 1, 0);
    for (int i = 0; i < n; ++i) {
        pre[i + 1] = pre[i] + nums[i];
    }
    return pre;
}

long long rangeSum(const vector<long long>& pre, int left, int right) {
    return pre[right + 1] - pre[left];
}
为什么前缀和数组长度通常是 n + 1？因为我们让 pre[i] 表示“前 i 个元素的和”，这样区间公式统一，不需要对 left=0 特判。

4.2 前缀和 + 哈希

如果题目要求“有多少个子数组和等于 k”，就不能只靠普通前缀和了，因为我们不是查询固定区间，而是在扫描过程中寻找以前出现过的某种前缀值。

// 子数组和等于 k 的个数
int subarraySum(vector<int>& nums, int k) {
    unordered_map<long long, int> cnt;
    cnt[0] = 1;
    long long pre = 0;
    int ans = 0;

    for (int x : nums) {
        pre += x;
        if (cnt.count(pre - k)) {
            ans += cnt[pre - k];
        }
        ++cnt[pre];
    }
    return ans;
}

这里的逻辑是：如果当前前缀和是 pre，要找一个之前的前缀和 old，使得 pre - old = k，那么就等价于找 old = pre - k 出现了多少次。

4.3 差分原理

前缀和擅长“区间查询”，差分擅长“区间修改”。如果我们要对很多区间做加法，直接逐个区间逐个位置修改会很慢。差分数组只记录“变化从哪里开始、在哪里结束”。

示意图：对区间 [l, r] 加 val，只在边界处做标记

// 对多个区间执行加法
vector<int> rangeAdd(int n, const vector<array<int, 3>>& ops) {
    vector<int> diff(n + 1, 0);

    for (auto &op : ops) {
        int l = op[0], r = op[1], val = op[2];
        diff[l] += val;
        if (r + 1 < n) {
            diff[r + 1] -= val;
        }
    }

    vector<int> ans(n, 0);
    if (n > 0) {
        ans[0] = diff[0];
    }
    for (int i = 1; i < n; ++i) {
        ans[i] = ans[i - 1] + diff[i];
    }
    return ans;
}

5. 哈希与二分：一个管“快查”，一个管“找边界”

5.1 哈希表原理

哈希表适合处理“是否存在、出现次数、第一次出现的位置、某个值对应的信息”这类问题。它的核心思想就是：把值映射到桶中，平均情况下用常数时间做查询。

// 两数之和
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> pos; // 值 -> 下标
    for (int i = 0; i < (int)nums.size(); ++i) {
        int need = target - nums[i];
        if (pos.count(need)) {
            return {pos[need], i};
        }
        pos[nums[i]] = i;
    }
    return {};
}
两数之和这类题最重要的观察不是“用哈希”，而是把条件 a + b = target 改写成“当我看到 a 时，只要知道 target-a 是否已经出现过”。这是一种很常见的“边扫描边配对”思想。

5.2 哈希做频次统计

bool isAnagram(const string& s, const string& t) {
    if (s.size() != t.size()) {
        return false;
    }

    vector<int> cnt(256, 0);
    for (unsigned char c : s) {
        ++cnt[c];
    }
    for (unsigned char c : t) {
        if (--cnt[c] < 0) {
            return false;
        }
    }
    return true;
}

如果字符集很小，可以直接用数组代替哈希表，速度通常更快，也更稳定。

5.3 二分查找原理

二分查找真正厉害的地方不是“在有序数组里找一个数”，而是“在一个满足单调性的空间里找边界”。只要答案随着某个参数变化呈现“前面不满足、后面满足”或相反的结构，就能二分。

示意图：二分不是找值，而是在单调区间里找边界

// 找第一个 >= target 的位置
int lowerBound(const vector<int>& a, int target) {
    int left = 0;
    int right = (int)a.size(); // 搜索区间 [left, right)

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (a[mid] >= target) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}
二分最重要的不是背模板，而是始终明确三件事：当前区间是什么形式、mid 属于哪一边、循环结束时 left / right 各表示什么。

6. 单调栈与矩阵：从一维延伸到更复杂场景

6.1 单调栈原理

单调栈特别适合解决这类问题：对于每个位置，想快速找到它左边或右边第一个比它大 / 小的元素。暴力做法是从每个位置向外找，复杂度会很高；单调栈通过维护“可能成为答案的候选人”，让每个元素只进栈出栈一次。

示意图：当新元素更大时，栈顶较小元素的答案就确定了

// 求每个位置右侧第一个更大元素的下标
vector<int> nextGreaterIndex(const vector<int>& nums) {
    int n = nums.size();
    vector<int> ans(n, -1);
    stack<int> st; // 存下标，栈内对应值单调递减

    for (int i = 0; i < n; ++i) {
        while (!st.empty() && nums[i] > nums[st.top()]) {
            ans[st.top()] = i;
            st.pop();
        }
        st.push(i);
    }
    return ans;
}

单调栈的思维重点在于：被弹出的元素以后再也不可能成为别人的更优候选，所以放心丢掉，这就是它能做到线性复杂度的原因。

6.2 二维数组与矩阵

二维数组本质上还是数组，只是多了一维坐标。很多一维思想都能迁移到二维，例如前缀和、边界遍历、原地旋转。

二维前缀和

如果一维前缀和解决的是“区间和”，二维前缀和解决的就是“子矩阵和”。定义 pre[i][j] 表示左上角到位置 (i-1, j-1) 的矩形和。

// 构建二维前缀和
vector<vector<long long>> build2DPrefix(const vector<vector<int>>& a) {
    int n = a.size();
    int m = n ? a[0].size() : 0;
    vector<vector<long long>> pre(n + 1, vector<long long>(m + 1, 0));

    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= m; ++j) {
            pre[i][j] = pre[i - 1][j] + pre[i][j - 1] - pre[i - 1][j - 1] + a[i - 1][j - 1];
        }
    }
    return pre;
}
二维前缀和公式里有一个“减回去”的步骤，是因为上方区域和左方区域相加时，左上角重叠部分被算了两次，所以要减掉一次。这本质上是容斥原理。

矩阵旋转

// n x n 方阵顺时针旋转 90 度
void rotateMatrix(vector<vector<int>>& mat) {
    int n = mat.size();

    for (int i = 0; i < n; ++i) {
        for (int j = i + 1; j < n; ++j) {
            swap(mat[i][j], mat[j][i]);
        }
    }

    for (int i = 0; i < n; ++i) {
        reverse(mat[i].begin(), mat[i].end());
    }
}

这个做法的原理是：顺时针旋转 90 度可以拆成“沿主对角线转置”再“每一行翻转”。本质上是在分解坐标变换。

7. 字符串核心知识点

字符串从算法角度看，本质上也是数组，只不过元素从整数变成字符。因此数组上的很多技巧都能直接复用：双指针、滑动窗口、哈希统计、前缀思想都很常见。

7.1 字符串常见操作模型

匹配

主串中找模式串，代表算法是 KMP。

统计

统计字符出现次数、最近位置、是否重复，常用数组或哈希。

结构

例如回文、公共前后缀、周期性，这类题更偏向利用字符串自身结构。

7.2 KMP 原理

KMP 解决的是“模式匹配中的重复回退问题”。朴素匹配一旦失配，通常会把模式串移到下一个起点重新开始；KMP 则利用模式串自身的前后缀关系，知道“已经匹配过的部分里，有哪些信息是可以复用的”。

示意图：KMP 的核心是失配后不让主串回退

vector<int> buildLPS(const string& p) {
    int n = p.size();
    vector<int> lps(n, 0);

    for (int i = 1, len = 0; i < n; ) {
        if (p[i] == p[len]) {
            lps[i] = ++len;
            ++i;
        } else if (len > 0) {
            len = lps[len - 1];
        } else {
            lps[i] = 0;
            ++i;
        }
    }
    return lps;
}

int kmpSearch(const string& s, const string& p) {
    if (p.empty()) {
        return 0;
    }

    vector<int> lps = buildLPS(p);
    for (int i = 0, j = 0; i < (int)s.size(); ) {
        if (s[i] == p[j]) {
            ++i;
            ++j;
            if (j == (int)p.size()) {
                return i - j;
            }
        } else if (j > 0) {
            j = lps[j - 1];
        } else {
            ++i;
        }
    }
    return -1;
}
lps[i] 表示：在模式串前缀 p[0...i] 中，最长的“真前缀 = 真后缀”的长度。这个信息告诉我们，失配后模式串应该跳到哪里。

7.3 回文串：中心扩展

回文串的特点是左右对称，所以一个自然的想法是：枚举“中心”，然后向两边扩展。每个回文串要么以一个字符为中心，要么以两个字符之间为中心。

示意图：回文有两种中心

string longestPalindrome(const string& s) {
    int n = s.size();
    if (n <= 1) {
        return s;
    }

    auto expand = [&](int left, int right) {
        while (left >= 0 && right < n && s[left] == s[right]) {
            --left;
            ++right;
        }
        return pair<int, int>{left + 1, right - 1};
    };

    int bestStart = 0;
    int bestLen = 1;

    for (int i = 0; i < n; ++i) {
        auto [l1, r1] = expand(i, i);
        auto [l2, r2] = expand(i, i + 1);

        if (r1 - l1 + 1 > bestLen) {
            bestStart = l1;
            bestLen = r1 - l1 + 1;
        }
        if (r2 - l2 + 1 > bestLen) {
            bestStart = l2;
            bestLen = r2 - l2 + 1;
        }
    }

    return s.substr(bestStart, bestLen);
}

7.4 字符串模拟

很多字符串题其实没有复杂算法，而是考察边界控制，例如：去除多余空格、反转单词、字符串加法、大整数乘法、解码表达式。做这类题的关键通常是分步骤处理，每一步只做一件事。

8. 高频模板与使用建议

有序数组去重

int removeDuplicates(vector<int>& nums) {
    if (nums.empty()) {
        return 0;
    }
    int slow = 1;
    for (int fast = 1; fast < (int)nums.size(); ++fast) {
        if (nums[fast] != nums[fast - 1]) {
            nums[slow++] = nums[fast];
        }
    }
    return slow;
}

合并两个有序数组

vector<int> mergeSorted(const vector<int>& a, const vector<int>& b) {
    vector<int> res;
    int i = 0, j = 0;
    while (i < (int)a.size() && j < (int)b.size()) {
        if (a[i] <= b[j]) {
            res.push_back(a[i++]);
        } else {
            res.push_back(b[j++]);
        }
    }
    while (i < (int)a.size()) res.push_back(a[i++]);
    while (j < (int)b.size()) res.push_back(b[j++]);
    return res;
}

旋转数组

void rotate(vector<int>& nums, int k) {
    int n = nums.size();
    if (n == 0) {
        return;
    }
    k %= n;
    reverse(nums.begin(), nums.end());
    reverse(nums.begin(), nums.begin() + k);
    reverse(nums.begin() + k, nums.end());
}

原理是“三次翻转”：整体翻转后，前 k 段和后 n-k 段的相对顺序刚好还原为目标状态。

二分模板

int firstTrue(int left, int right) { // [left, right)
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (check(mid)) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    return left;
}

任何“找第一个满足条件的位置”的问题，几乎都能套这个结构，前提是 check 具有单调性。

8.1 什么时候选哪一种

  • 连续区间最值、最短长度、无重复子串：滑动窗口
  • 区间和、子矩阵和：前缀和 / 二维前缀和
  • 大量区间加减：差分
  • 出现过、配对、频次统计：哈希
  • 原地删除、去重、反转：双指针
  • 有序数组边界查找：二分查找
  • 右侧第一个更大元素、柱状图面积：单调栈
  • 主串匹配模式串：KMP
  • 最长回文子串：中心扩展

9. 常见错误与调试建议

边界写错

最常见的问题是区间定义不统一。比如二分到底是 [l, r] 还是 [l, r)，窗口长度到底是 right-left+1 还是别的形式。建议固定一套写法，不要混用。

忘记空输入

数组或字符串可能为空，矩阵可能没有列，访问 nums[0] 或 a[0].size() 前要先判断。

整数溢出

前缀和、区间和、乘法累加尽量使用 long long。尤其是题目数据范围大时，int 很容易溢出。

字符范围想当然

若题目不是明确只有小写字母，就不要把频次数组默认开成 26。ASCII 常用 128 或 256，Unicode 则需要更谨慎的处理。

窗口不满足适用条件

滑动窗口依赖某种单调性。比如“最短子数组和至少为 k”只有在元素非负时才适合普通窗口，包含负数后就要考虑其他方法。

调试时只看答案，不看状态

这类题更适合打印关键状态：left、right、sum、栈内容、哈希计数、pre 数组。只盯着最终答案，往往很难定位 bug。

调试数组和字符串题时，一个非常有效的方法是：手写一个很小的例子，跟着指针或状态变量一步一步模拟。很多边界问题在纸上走两轮就能看出来。

10. 学习建议：如何把套路变成能力

真正的提升不在于背多少题，而在于看到题目时，能快速判断它属于哪种模型。建议的学习顺序如下：

  1. 先把数组遍历、反转、删除、去重这类基础操作写熟。
  2. 重点掌握双指针与滑动窗口，因为它们是数组和字符串里最常见的线性优化手段。
  3. 接着学习前缀和、差分、哈希，这三类方法常常一起出现。
  4. 再掌握二分和单调栈，理解“边界”和“单调性”这两个关键词。
  5. 最后学习 KMP、回文串、二维前缀和等更强调结构理解的内容。
一道题做完后，最好追问自己三件事：为什么暴力慢？优化依赖的关键性质是什么？如果数据范围或条件稍微变化，这个方法还成立吗？这三个问题能帮助你真正吃透原理。
