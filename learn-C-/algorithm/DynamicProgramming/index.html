1. 什么是动态规划

动态规划，简称 DP，本质上是在解决这样一类问题： 一个大问题可以拆成若干个更小的子问题，而这些子问题会被重复计算。 如果我们把每个子问题的答案存下来，后面直接复用，就能把指数级暴力递归优化成多项式时间。

所以 DP 不是一种固定模板，而是一种“避免重复计算”的思维方式。 它通常适用于具有两个典型特征的问题：

  • 最优子结构：原问题的最优解可以由子问题的最优解组合得到。
  • 重叠子问题：同一个子问题会在递归中被多次求解。
很多题一开始看起来像搜索或递归，但只要你发现“同样的状态会反复出现”，就该考虑 DP。

1.1 记忆化搜索与递推 DP

记忆化搜索

写法接近递归，适合从“暴力搜索”自然改造。

递推 DP

更适合面试和竞赛中直接写表格或数组状态转移。

1.2 一个最经典的例子：斐波那契

int fib(int n) {
    if (n <= 1) return n;
    vector<int> dp(n + 1);
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}

这里的 dp[i] 表示第 i 个斐波那契数。状态转移很自然： dp[i] = dp[i - 1] + dp[i - 2]。 斐波那契虽然简单，但几乎包含了 DP 的全部骨架：状态、转移、初始化、遍历顺序。

2. 动态规划如何设计

多数人学 DP 卡住，不是因为代码难，而是因为不知道状态怎么定义。一个稳定的思考顺序是：

  1. 先问自己：原问题能不能拆成更小的同类问题？
  2. 定义 dp[i]、dp[i][j] 或更复杂状态到底表示什么。
  3. 思考最后一步是怎么来的，从而写出状态转移。
  4. 确定哪些状态是已知的，也就是初始化。
  5. 确定必须按什么顺序推，保证用到的子状态已经算好。

2.1 “最后一步”思想

很多 DP 的转移都可以从“最后一步做了什么”入手。例如爬楼梯到第 i 级， 最后一步要么从 i-1 迈 1 步过来，要么从 i-2 迈 2 步过来。 于是答案就是这两种情况之和。 

int climbStairs(int n) {
    vector<int> dp(n + 1);
    dp[0] = 1;
    dp[1] = 1;
    for (int i = 2; i <= n; ++i) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}

状态定义

例如 dp[i] 表示前 i 个元素的最优值，或到位置 i 的方案数。

状态转移

考虑当前决策后，状态如何从已知子问题转移而来。

初始化

最小规模问题的答案必须先给出来，否则大状态无法推出。

遍历顺序

保证转移所依赖的状态已经先被计算出来。

2.2 空间优化为什么可行

如果一个状态只依赖前面有限几个状态，就没必要把整张表都留着。比如斐波那契只依赖前两项， 那就可以把 O(n) 空间优化成 O(1)。 

int fibOptimized(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; ++i) {
        int c = a + b;
        a = b;
        b = c;
    }
    return b;
}
空间优化的前提是你非常清楚当前状态依赖谁。否则一旦覆盖过早，转移就会出错。

3. 线性 DP 与序列类问题

3.1 打家劫舍

这是“选或不选”型 DP 的代表。到第 i 个房子时，要么不偷它，答案就是前一个房子的最优解； 要么偷它，那么前一个房子不能偷，答案是 dp[i-2] + nums[i]。 

int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    if (n == 1) return nums[0];

    vector<int> dp(n);
    dp[0] = nums[0];
    dp[1] = max(nums[0], nums[1]);

    for (int i = 2; i < n; ++i) {
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i]);
    }
    return dp[n - 1];
}

不选当前

当前不拿，答案直接继承前一个状态。

选当前

当前拿了，就必须从隔一个的位置转移过来。

3.2 最大子数组和

这题的状态可以定义为：dp[i] 表示“必须以 i 结尾”的最大子数组和。 那么要么把前面的连续段接上，要么从当前位置重新开始。 

int maxSubArray(vector<int>& nums) {
    int cur = nums[0], ans = nums[0];
    for (int i = 1; i < (int)nums.size(); ++i) {
        cur = max(nums[i], cur + nums[i]);
        ans = max(ans, cur);
    }
    return ans;
}

3.3 最长递增子序列 LIS

经典 O(n^2) DP 定义是： dp[i] 表示以 nums[i] 结尾的最长递增子序列长度。 枚举所有 j < i，如果 nums[j] < nums[i]，就可以从 j 转移到 i。 

int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    vector<int> dp(n, 1);
    int ans = 1;

    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < i; ++j) {
            if (nums[j] < nums[i]) {
                dp[i] = max(dp[i], dp[j] + 1);
            }
        }
        ans = max(ans, dp[i]);
    }
    return ans;
}

4. 背包 DP

背包是 DP 中最重要的模型之一。它的本质是： 在容量限制下，从若干物品中做选择，使得总价值最大或方案数满足要求。

4.1 0/1 背包

每个物品只能选 0 次或 1 次。设 dp[j] 表示容量恰为或不超过 j 时的最大价值， 对每个物品都尝试“选”或“不选”。 

int zeroOneKnapsack(vector<int>& weight, vector<int>& value, int bag) {
    vector<int> dp(bag + 1, 0);
    int n = weight.size();

    for (int i = 0; i < n; ++i) {
        for (int j = bag; j >= weight[i]; --j) {
            dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
        }
    }
    return dp[bag];
}

这里容量必须从大到小枚举，否则同一件物品会被重复使用，变成完全背包。

0/1 背包

每个物品最多用一次，容量通常倒序枚举。

完全背包

每个物品可无限次使用，容量通常正序枚举。

多重背包

每个物品可用有限多次，可转成多件 0/1 物品或用优化技巧。

4.2 完全背包

int completeKnapsack(vector<int>& weight, vector<int>& value, int bag) {
    vector<int> dp(bag + 1, 0);
    int n = weight.size();

    for (int i = 0; i < n; ++i) {
        for (int j = weight[i]; j <= bag; ++j) {
            dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
        }
    }
    return dp[bag];
}

4.3 组合数 / 方案数类型

有些背包不是求最大值，而是求多少种方法，比如零钱兑换 II。此时转移从 max 变成“累加方案数”。 

int change(int amount, vector<int>& coins) {
    vector<int> dp(amount + 1, 0);
    dp[0] = 1;

    for (int coin : coins) {
        for (int j = coin; j <= amount; ++j) {
            dp[j] += dp[j - coin];
        }
    }
    return dp[amount];
}
背包题特别容易混淆“组合”和“排列”。外层循环先物品还是先容量，会影响答案含义。

5. 其他常见 DP 模型

5.1 二维 DP：最长公共子序列 LCS

设 dp[i][j] 表示字符串 s[0..i-1] 和 t[0..j-1] 的最长公共子序列长度。 如果当前字符相等，就从左上角加一；否则从上方和左方取最大值。 

int longestCommonSubsequence(string text1, string text2) {
    int n = text1.size(), m = text2.size();
    vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));

    for (int i = 1; i <= n; ++i) {
        for (int j = 1; j <= m; ++j) {
            if (text1[i - 1] == text2[j - 1]) {
                dp[i][j] = dp[i - 1][j - 1] + 1;
            } else {
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
            }
        }
    }
    return dp[n][m];
}

字符相等

dp[i][j] = dp[i-1][j-1] + 1

字符不等

dp[i][j] = max(dp[i-1][j], dp[i][j-1])

5.2 区间 DP：回文子串 / 戳气球 / 矩阵连乘

区间 DP 的状态一般是 dp[l][r]，表示区间 [l, r] 的答案。 关键在于枚举区间长度，保证短区间先算出来，再推长区间。 

int countSubstrings(string s) {
    int n = s.size(), ans = 0;
    vector<vector<bool>> dp(n, vector<bool>(n, false));

    for (int len = 1; len <= n; ++len) {
        for (int l = 0; l + len - 1 < n; ++l) {
            int r = l + len - 1;
            if (s[l] == s[r]) {
                if (len <= 2) dp[l][r] = true;
                else dp[l][r] = dp[l + 1][r - 1];
            }
            if (dp[l][r]) ++ans;
        }
    }
    return ans;
}

5.3 树形 DP

树形 DP 常见于二叉树或一般树上。典型思路是： 每个节点返回若干状态，父节点根据子节点状态进行合并。 比如打家劫舍 III 中，每个节点维护“偷这个点”和“不偷这个点”两种情况。 

pair<int, int> dfs(TreeNode* root) {
    if (!root) return {0, 0};
    auto [l0, l1] = dfs(root->left);
    auto [r0, r1] = dfs(root->right);

    int rob = root->val + l0 + r0;
    int notRob = max(l0, l1) + max(r0, r1);
    return {notRob, rob};
}

5.4 状态机 DP

某些题里，状态不只是位置，还包括“当前处于什么状态”。最典型的是股票买卖问题： 每一天你可能处于持有股票、不持有股票、冷冻期等状态，然后按状态转移。 

int maxProfit(vector<int>& prices) {
    int hold = -prices[0];
    int sold = 0;
    for (int i = 1; i < (int)prices.size(); ++i) {
        hold = max(hold, -prices[i]);
        sold = max(sold, hold + prices[i]);
    }
    return sold;
}

6. 常用模板、复杂度与总结

一维 DP 模板

vector<int> dp(n + 1, 0);
dp[0] = base;
for (int i = 1; i <= n; ++i) {
    dp[i] = ... dp[i - 1] ...
}

二维 DP 模板

vector<vector<int>> dp(n + 1, vector<int>(m + 1, 0));
for (int i = 1; i <= n; ++i) {
    for (int j = 1; j <= m; ++j) {
        dp[i][j] = ...;
    }
}

记忆化搜索模板

int dfs(int state) {
    if (memo.count(state)) return memo[state];
    int ans = ...;
    return memo[state] = ans;
}

6.1 复杂度怎么分析

模型 时间复杂度 空间复杂度
一维线性 DP 通常 O(n) O(n) 或优化到 O(1)
二维字符串 DP 通常 O(nm) O(nm)
0/1 背包 O(n * bag) O(bag)
区间 DP 常见为 O(n^2) 或 O(n^3) O(n^2)

6.2 刷题时最常见的错误

  • 状态定义含糊，不知道 dp[i] 到底表示什么。
  • 初始化不完整，导致后续转移建立在错误边界上。
  • 遍历顺序写反，使用了还没计算出来的状态。
  • 背包题把倒序和正序枚举混用，导致物品使用次数错了。
  • 看见最优解就盲目上 DP，但问题实际上没有合适的最优子结构。
学 DP 最有效的方法，不是背每道题的代码，而是反复练这四件事： 定义状态、找最后一步、写转移、定遍历顺序。 只要这套思路稳定下来，很多题都只是模型的变形。

背诵版总结

  • DP 是把重复子问题的答案存下来复用。
  • 先定义状态，再谈转移。
  • 转移通常从“最后一步”或“当前决策”出发。
  • 初始化决定了整个 DP 的边界是否正确。
  • 遍历顺序必须满足依赖关系。
  • 空间优化前要先明确依赖范围。
  • 背包题关键在于物品是否可重复使用，以及循环顺序。

推荐刷题顺序

  1. 先练斐波那契、爬楼梯、打家劫舍这类一维 DP。
  2. 再练最大子数组和、LIS、编辑距离、LCS。
  3. 然后系统练 0/1 背包、完全背包、零钱兑换。
  4. 最后再练区间 DP、树形 DP、状态机 DP。
