## 一、I/O 多路复用深入

### 1.1 select / poll / epoll 对比

| 特性 | select | poll | epoll |
|------|--------|------|-------|
| 最大 FD 数 | 1024（FD_SETSIZE） | 无限制 | 无限制 |
| 时间复杂度 | O(n) 每次全量扫描 | O(n) 每次全量扫描 | O(1) 仅活跃 FD |
| 内核到用户空间 | 每次调用拷贝全量 fd_set | 每次拷贝全量 pollfd 数组 | epoll_wait 仅返回就绪 fd 列表，无需全量拷贝 |
| 触发模式 | 仅水平触发（LT） | 仅水平触发（LT） | LT + 边缘触发（ET） |
| 适用场景 | 跨平台兼容 | FD 数中等 | 高并发、大量连接 |

### 1.2 epoll 两种触发模式

**水平触发（Level Triggered, LT）**：只要 FD 就绪，每次 `epoll_wait` 都会通知。默认模式，编程简单，不易遗漏事件。

**边缘触发（Edge Triggered, ET）**：仅在 FD 状态变化时通知一次。必须使用非阻塞 I/O，且需要循环读写直到 `EAGAIN`/`EWOULDBLOCK`。性能更高，适合高吞吐场景。

```cpp
// ET 模式下的读处理伪代码
while (true) {
    ssize_t n = read(fd, buf, sizeof(buf));
    if (n > 0) {
        // 处理数据
    } else if (n == 0) {
        // 对端关闭
        close(fd);
        break;
    } else {
        if (errno == EAGAIN || errno == EWOULDBLOCK) break;  // 数据读完
        // 其他错误处理
        break;
    }
}
```
