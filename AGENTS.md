# cppinter — 中国 C++ 社招面试资料库

## 仓库性质

这是一个 **C++ 社招面试准备资料库**（中国招聘市场），**不是代码仓库**。所有内容已合并为单一文件 `ALL_IN_ONE.md`（~59,309 行），原始各目录下的 `README.md` 仍保留但内容已统一归入 `ALL_IN_ONE.md`。

## 已知关键约束

1. **全量合并至 ALL_IN_ONE.md** — 所有 8 个目录的 README.md 已按顺序合并为单一文件（~59,309 行），各节之间以 `---` 分隔，顶部有目录索引。
2. **所有跨目录链接断裂** — 原始 `.md` 文件已合并删除，URL 编码的相对链接（`problems__*.md`、`docs__*.md` 等）全部无效。修改时只应写纯文本标题。
3. **非 C++ 内容混入** — "面试鸭"章节是 React+Node.js 全文，如果清理要谨慎确认。
4. **百度网盘链接** — `编程语言基础/README.md` 有大量链接，涉及版权风险。
5. **内容来自多个仓库合并** — 可能存在重复、冲突、分类不当（如栈/队列出现在操作系统目录）。
6. **STL 章节是 API 搬运** — 早期来自 cplusplus.com，深层原理在后补章节中。
7. **单例代码示例有隐患** — 懒汉式非线程安全、使用 `0` 而非 `nullptr`、缺拷贝禁用。已加警告注释。

## 修改规范

- 所有知识性修改必须核实（优先用 cppreference 而非 CSDN）
- 新增内容使用 `---` 分隔，保持原有 `#` / `###` 标题层级
- C++ 版本标注要精确（如 random_shuffle C++14 废弃 C++17 移除）
- 不要删除旧文件名式标题（如 `# problems__xxx.md`），它们是合并来源标识
- 对不安全代码示例（单例、过时 API）补充注释而非直接修改代码
