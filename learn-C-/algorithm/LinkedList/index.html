1. 链表的基础概念

链表是一种通过指针把节点串起来的线性结构。与数组不同，链表中的节点在内存中不要求连续， 每个节点只需要知道“下一个节点在哪里”即可。

所以链表的核心不是“下标”，而是“连接关系”。算法题里你真正操作的不是值，而是 next 和 prev 这些指针。

链表题的第一原则是： 先想清楚谁指向谁，再写代码。 一旦指针修改顺序错了，很容易把后续节点丢失。
  • 头节点：链表的入口。
  • 当前节点：指针当前指向的节点。
  • 后继节点：node->next。
  • 前驱节点：当前节点前面的节点，单链表通常需要额外记录。
  • 哑节点 dummy：人为添加的头前节点，用来统一边界处理。

1.1 单链表与双向链表

单链表

每个节点只知道下一个节点，节省空间，但删除时通常要先找到前驱。

双向链表

每个节点同时知道前驱和后继，插删更灵活，是 LRU 的常见基础结构。

哑节点 dummy

当头节点也可能被删或被插到前面时，dummy 能让边界情况和普通情况统一。

2. 链表题的指针操作本质

大多数链表题本质上都在做下面几件事：找到某个位置、断开旧连接、建立新连接、保存可能丢失的后继节点。 只要把这几个动作拆开，代码就会变得稳定。

2.1 删除节点

删除某节点的本质不是“擦掉它”，而是让它的前驱跳过它，直接指向它的后继。 所以删除时最关键的是前驱节点。 

ListNode* removeElements(ListNode* head, int val) {
    ListNode dummy(0);
    dummy.next = head;
    ListNode* cur = &dummy;

    while (cur->next) {
        if (cur->next->val == val) {
            cur->next = cur->next->next;
        } else {
            cur = cur->next;
        }
    }
    return dummy.next;
}

2.2 插入节点

在链表中间插入一个节点时，最安全的顺序是： newNode->next = prev->next，然后再 prev->next = newNode。 先接上后半段，再接前半段，避免后继丢失。 

void insertAfter(ListNode* prev, ListNode* node) {
    if (!prev || !node) return;
    node->next = prev->next;
    prev->next = node;
}
链表操作里最常见的错误，是还没保存 next 就把指针改掉，结果后面的节点全丢了。 所以遇到“重连”类操作时，经常先写一句：ListNode* nxt = cur->next;

2.3 为什么 dummy 节点这么常用

  • 头节点被删除时，不需要单独写分支。
  • 在头部插入节点时，逻辑和中间插入完全一致。
  • 返回答案时统一写成 dummy.next。

3. 快慢指针思想

快慢指针是链表题里最重要的技巧之一。它的本质是让两个指针以不同速度前进， 通过它们的相对位置关系来得到中点、环、倒数第 k 个节点等信息。

3.1 找中点

让 slow 每次走一步，fast 每次走两步。 当 fast 到末尾时，slow 恰好在中间。 

ListNode* middleNode(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
    }
    return slow;
}

判断有环

如果有环，快指针最终一定会在环里追上慢指针。

找倒数第 k 个

两个指针保持固定间距，这是双指针在链表上的经典应用。

bool hasCycle(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;
    }
    return false;
}

ListNode* getKthFromEnd(ListNode* head, int k) {
    ListNode* fast = head;
    ListNode* slow = head;
    while (k-- && fast) fast = fast->next;
    while (fast) {
        fast = fast->next;
        slow = slow->next;
    }
    return slow;
}

4. 链表经典题型与实现

4.1 反转链表

反转链表是链表题的基本功。核心就是把每个节点的 next 指向前一个节点。 这个过程必须先保存原来的后继，否则后面的链会断掉。

反转过程

当前节点回头指向前驱，链表方向就逐步反过来了。

ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* cur = head;

    while (cur) {
        ListNode* nxt = cur->next;
        cur->next = prev;
        prev = cur;
        cur = nxt;
    }
    return prev;
}

4.2 合并两个有序链表

这题和归并排序的 merge 很像。每次比较两个链表的当前节点，把较小的接到答案链表后面。 

ListNode* mergeTwoLists(ListNode* l1, ListNode* l2) {
    ListNode dummy(0);
    ListNode* tail = &dummy;

    while (l1 && l2) {
        if (l1->val < l2->val) {
            tail->next = l1;
            l1 = l1->next;
        } else {
            tail->next = l2;
            l2 = l2->next;
        }
        tail = tail->next;
    }

    tail->next = l1 ? l1 : l2;
    return dummy.next;
}

4.3 删除倒数第 N 个节点

先让快指针比慢指针领先 n 步，再同步前进。为了删除方便，通常让慢指针停在目标节点的前驱， 所以一般从 dummy 开始走。 

ListNode* removeNthFromEnd(ListNode* head, int n) {
    ListNode dummy(0);
    dummy.next = head;
    ListNode* fast = &dummy;
    ListNode* slow = &dummy;

    for (int i = 0; i < n; ++i) fast = fast->next;

    while (fast->next) {
        fast = fast->next;
        slow = slow->next;
    }

    slow->next = slow->next->next;
    return dummy.next;
}

4.4 判断链表是否有环，并找到入环点

快慢指针第一次相遇后，把其中一个指针放回头节点，然后两者每次都走一步。 它们再次相遇的地方，就是入环点。

入环点原理

第一次相遇点不一定是入环点，但二次同步前进一定会在入环点相遇。

ListNode* detectCycle(ListNode* head) {
    ListNode* slow = head;
    ListNode* fast = head;

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            ListNode* p = head;
            while (p != slow) {
                p = p->next;
                slow = slow->next;
            }
            return p;
        }
    }
    return nullptr;
}

4.5 两个链表的第一个相交节点

让两个指针分别从两条链表出发，走到末尾后切换到另一条链表头部。 这样两者总路程相等，最终会在相交点相遇；如果不相交，就一起走到空节点。 

ListNode* getIntersectionNode(ListNode* headA, ListNode* headB) {
    ListNode* p = headA;
    ListNode* q = headB;

    while (p != q) {
        p = p ? p->next : headB;
        q = q ? q->next : headA;
    }
    return p;
}

4.6 判断回文链表

常见做法是：快慢指针找中点，反转后半段，再从头和后半段头同时比较。 这题把多个链表模板拼到了一起，是很典型的综合题。 

bool isPalindrome(ListNode* head) {
    if (!head || !head->next) return true;

    ListNode* slow = head;
    ListNode* fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
    }

    ListNode* prev = nullptr;
    while (slow) {
        ListNode* nxt = slow->next;
        slow->next = prev;
        prev = slow;
        slow = nxt;
    }

    ListNode* p1 = head;
    ListNode* p2 = prev;
    while (p2) {
        if (p1->val != p2->val) return false;
        p1 = p1->next;
        p2 = p2->next;
    }
    return true;
}

5. 双向链表与 LRU 基础

双向链表在算法题中最常见的场景是 LRU 缓存。因为 LRU 需要支持： 某个节点快速删除、快速插到头部、快速拿到尾部最久未使用节点。 这正是双向链表擅长的事情。

5.1 双向链表的删除与头插

双向链表的删除本质是让一个节点的前驱和后继重新连起来： node->prev->next = node->next， node->next->prev = node->prev。 

void removeNode(DListNode* node) {
    node->prev->next = node->next;
    node->next->prev = node->prev;
}

void addToFront(DListNode* head, DListNode* node) {
    node->next = head->next;
    node->prev = head;
    head->next->prev = node;
    head->next = node;
}

5.2 LRU 为什么是 哈希表 + 双向链表

哈希表的作用

根据 key 在 O(1) 时间定位到对应节点。

双向链表的作用

在 O(1) 时间删除任意节点，并把最近访问节点移到头部。

尾部节点的意义

尾部就是最久未使用节点，容量满时直接淘汰尾部前一个节点。

class LRUCache {
private:
    int capacity;
    unordered_map<int, DListNode*> mp;
    DListNode* head;
    DListNode* tail;

    void removeNode(DListNode* node) {
        node->prev->next = node->next;
        node->next->prev = node->prev;
    }

    void addToFront(DListNode* node) {
        node->next = head->next;
        node->prev = head;
        head->next->prev = node;
        head->next = node;
    }

public:
    LRUCache(int cap) : capacity(cap) {
        head = new DListNode(0, 0);
        tail = new DListNode(0, 0);
        head->next = tail;
        tail->prev = head;
    }

    int get(int key) {
        if (!mp.count(key)) return -1;
        DListNode* node = mp[key];
        removeNode(node);
        addToFront(node);
        return node->val;
    }

    void put(int key, int value) {
        if (mp.count(key)) {
            DListNode* node = mp[key];
            node->val = value;
            removeNode(node);
            addToFront(node);
            return;
        }

        if ((int)mp.size() == capacity) {
            DListNode* node = tail->prev;
            removeNode(node);
            mp.erase(node->key);
        }

        DListNode* node = new DListNode(key, value);
        addToFront(node);
        mp[key] = node;
    }
};

6. 常用模板、复杂度与总结

反转模板

ListNode* prev = nullptr;
ListNode* cur = head;
while (cur) {
    ListNode* nxt = cur->next;
    cur->next = prev;
    prev = cur;
    cur = nxt;
}

快慢指针模板

ListNode* slow = head;
ListNode* fast = head;
while (fast && fast->next) {
    slow = slow->next;
    fast = fast->next->next;
}

dummy 模板

ListNode dummy(0);
dummy.next = head;
ListNode* cur = &dummy;

6.1 常见误区

  • 修改 cur->next 前没有先保存后继节点。
  • 删除头节点时没用 dummy，导致单独写很多边界分支。
  • 快慢指针循环条件写错，空指针解引用。
  • 链表是否有环、找中点、找倒数第 k 个节点这些模板混用时没分清目标。
  • 双向链表删除节点时只改了一边，导致链表结构损坏。

6.2 复杂度分析

操作 时间复杂度 空间复杂度
遍历链表 O(n) O(1)
反转链表 O(n) O(1)
快慢指针找中点 / 判断有环 O(n) O(1)
LRU 的 get / put 平均 O(1) O(capacity)
链表题真正要练的不是记住题目，而是把这些动作练熟： 找前驱、保存后继、重连 next、快慢指针保持间距、dummy 统一边界。 这些动作一旦熟练，很多题都只是模板变形。

背诵版总结

  • 链表是靠指针连接，不靠下标访问。
  • 删除的本质是前驱跳过当前节点。
  • 插入的本质是先接后半段，再接前半段。
  • 反转时必须先保存 nxt。
  • 快慢指针解决中点、环、倒数第 k 个等问题。
  • dummy 节点能统一头节点边界处理。
  • LRU = 哈希表 + 双向链表。

刷题顺序建议

  1. 先练删除、反转、合并两个有序链表。
  2. 再练删除倒数第 N 个、找中点、回文链表。
  3. 然后练环检测、入环点、相交链表。
  4. 最后练双向链表与 LRU。
