==================================================
二、语言基础必会
==================================================

1. 指针和引用的区别
- 指针本质是变量，存放地址，可以为空，可以修改指向。
- 引用本质上是别名，定义时必须初始化，通常不能为空，也不能改绑。
- 面试表达：引用更像“语法级别的别名”，常用于参数传递和返回值，避免拷贝。

2. const 的常见形式
- const int a：a 不可改。
- const int* p：不能通过 p 修改其指向内容。
- int* const p：p 自己不能改指向。
- const int* const p：内容和指向都不能改。
- 成员函数后加 const：承诺不修改对象的逻辑状态。

3. static 的作用
- 修饰局部变量：生命周期延长到程序结束，但作用域不变。
- 修饰全局变量/函数：限制链接范围到当前编译单元。
- 修饰类成员变量：属于类，不属于对象。
- 修饰类成员函数：没有 this 指针，只能访问静态成员。

4. inline 的作用
- 早期主要用于建议编译器内联展开。
- 现代编译器是否内联主要自己决定。
- 更重要的意义是允许函数在多个翻译单元中定义而不违反 ODR。

5. C 和 C++ 的区别
- C 是面向过程，C++ 支持面向对象和泛型。
- C++ 有构造/析构、重载、模板、异常、RAII、STL。
- C++ 更强调类型系统和资源管理抽象。

6. struct 和 class 的区别
- 默认访问权限不同：struct 默认 public，class 默认 private。
- 默认继承权限不同：struct 默认 public 继承，class 默认 private 继承。
- 除此之外本质能力一致。

==================================================
三、面向对象与多态
==================================================

1. 封装、继承、多态
- 封装：隐藏实现细节，暴露稳定接口。
- 继承：复用基类接口和实现。
- 多态：同一接口，不同对象表现不同。

2. 重载、重写、隐藏
- 重载：同一作用域，同名函数参数列表不同。
- 重写：派生类覆盖基类虚函数，函数签名要匹配。
- 隐藏：派生类定义同名函数，会屏蔽基类同名函数。

3. 虚函数的实现原理
- 含虚函数的类通常有虚表指针 vptr。
- 虚表中存放虚函数地址。
- 通过基类指针/引用调用虚函数时发生动态绑定。
- 非虚函数是静态绑定。

4. 为什么析构函数常常要设为虚函数
- 如果通过基类指针 delete 派生类对象，而基类析构不是虚函数，会导致派生类析构不被调用，资源泄漏。
- 经验：有多态基类时，析构函数通常必须是虚函数。

5. 纯虚函数和抽象类
- 纯虚函数写法：virtual void f() = 0;
- 含纯虚函数的类是抽象类，不能实例化。
- 抽象类常用来定义接口。

6. 为什么不建议在构造和析构里调用虚函数
- 构造期间派生部分尚未完成。
- 析构期间派生部分已经销毁。
- 此时调用不会表现出完整多态行为，容易出现误解和 bug。

==================================================
四、拷贝控制、RAII、Rule of 3/5/0
==================================================

1. 浅拷贝和深拷贝
- 浅拷贝只复制指针值，多个对象共享同一块资源，容易重复释放。
- 深拷贝会复制资源本体，每个对象拥有独立资源。

2. Rule of 3
- 如果类需要自定义析构函数、拷贝构造、拷贝赋值中的一个，通常另外两个也要考虑自定义。
- 因为类往往自己管理资源。

3. Rule of 5
- C++11 后还要考虑移动构造和移动赋值。
- 当类自己管理资源时，通常要成套考虑这 5 个特殊成员函数。

4. Rule of 0
- 最理想的设计是不要自己手写资源管理，把资源交给已有 RAII 类型，如 string、vector、unique_ptr。
- 这样特殊成员函数可以依赖编译器生成，代码更安全。

5. 什么是 RAII
- Resource Acquisition Is Initialization。
- 资源在对象构造时获取，在析构时释放。
- 核心价值：自动释放资源，异常安全，减少泄漏。
- 常见资源：堆内存、文件句柄、锁、socket、数据库连接。

6. RAII 的典型例子
- std::unique_ptr 管理堆对象。
- std::lock_guard 在构造时加锁，析构时解锁。
- std::fstream 在对象析构时关闭文件。

面试回答模板
- RAII 是现代 C++ 最核心的资源管理思想。资源绑定到对象生命周期上，利用析构自动清理资源，从而减少泄漏并提高异常安全性。智能指针和锁封装都是典型应用。

==================================================
五、move 语义、右值引用、完美转发
==================================================

1. 左值和右值
- 左值：有名字、可取地址、可持久存在的对象。
- 右值：临时对象、将亡值，通常用于触发移动语义。

2. 什么是右值引用
- 写法：T&&
- 主要用于绑定右值，支持移动语义和完美转发。

3. 什么是 move 语义
- 对资源所有权进行“转移”而不是“复制”。
- 典型收益：减少堆内存分配和深拷贝成本。
- std::move 本身不移动，只是把对象转成右值表达式，真正是否移动取决于目标类型是否有移动操作。

4. move 后对象还能用吗
- 能析构、能重新赋值。
- 但其值只保证“有效但未指定”。
- 面试时不要说“move 后对象一定为空”，这是错误的。

5. 拷贝构造和移动构造的区别
- 拷贝构造复制资源。
- 移动构造窃取资源所有权，并让源对象进入可析构但未指定状态。

6. 什么是完美转发
- 在模板中保持参数原本的值类别和 cv 属性传递下去。
- 典型写法：std::forward<T>(arg)

7. std::move 和 std::forward 的区别
- std::move：无条件转成右值。
- std::forward：条件转发，只在模板推导场景中保留原始值类别。

常见追问
- 为什么 named rvalue reference 本身是左值？
  因为表达式有名字，按语言规则它是左值，所以再次传递时常要 std::move。

==================================================
六、智能指针
==================================================

1. unique_ptr
- 独占所有权，不可拷贝，可移动。
- 开销小，默认首选。
- 适合表达唯一拥有关系。

2. shared_ptr
- 共享所有权，内部有引用计数。
- 最后一个 shared_ptr 销毁时释放资源。
- 有额外计数开销，滥用会影响性能和语义清晰度。

3. weak_ptr
- 不增加共享计数，用来观察 shared_ptr 管理的对象。
- 主要用途：打破循环引用。

4. 为什么不推荐到处用 shared_ptr
- 所有权语义不清晰。
- 有原子计数成本。
- 容易形成循环引用。
- 在不需要共享时，unique_ptr 更清晰更高效。

5. make_unique 和 make_shared 的优点
- 异常安全更好。
- 代码更简洁。
- make_shared 通常会把控制块和对象一起分配，减少一次分配。

6. shared_ptr 循环引用
- A 持有 B 的 shared_ptr，B 持有 A 的 shared_ptr。
- 引用计数永不为 0，导致泄漏。
- 常用 weak_ptr 解决。

==================================================
七、new/delete、malloc/free、内存分区
==================================================

1. new/delete 和 malloc/free 的区别
- new/delete 是 C++ 运算符；malloc/free 是 C 库函数。
- new 分配内存后会调用构造函数；delete 会调用析构函数。
- malloc/free 只做原始内存分配与释放，不负责对象构造和析构。
- new 失败默认抛 bad_alloc；malloc 失败返回 nullptr。

2. delete 和 delete[] 的区别
- delete 用于单对象。
- delete[] 用于数组。
- 不匹配会导致未定义行为。

3. 程序内存布局
- 代码段
- 全局/静态存储区
- 堆
- 栈
- 常量区（实现相关，通常和只读数据相关）

4. 栈和堆的区别
- 栈由编译器/运行时自动管理，分配释放快。
- 堆由程序员或分配器管理，灵活但成本更高。
- 栈空间通常较小，堆空间相对大。

==================================================
八、STL 容器高频问题
==================================================

1. vector、list、deque 区别
- vector：连续内存，随机访问快，尾插尾删快，中间插入删除慢。
- list：双向链表，任意位置插删快，不支持随机访问，缓存局部性差。
- deque：分段连续，头尾插删都较快，随机访问支持但局部性通常不如 vector。

2. map 和 unordered_map 的区别
- map：红黑树，键有序，查改删一般 O(log n)。
- unordered_map：哈希表，平均 O(1)，最坏 O(n)，键无序。
- 若需要有序遍历、范围查询，用 map。
- 若追求平均查找性能，用 unordered_map。

3. set 和 unordered_set 的区别
- 原理和 map vs unordered_map 类似。

4. vector 扩容机制
- 容量不足时重新申请更大内存，把旧元素搬过去。
- 扩容倍数与实现有关，常见是 1.5 倍或 2 倍附近。
- 扩容会导致迭代器、指针、引用失效。

5. 迭代器失效
- vector 扩容后几乎所有迭代器失效。
- list 插入删除通常不会导致其他节点迭代器失效，但被删节点失效。
- unordered_map rehash 后迭代器可能失效。
- 面试中常考：erase 后返回值怎么用，边遍历边删除怎么写。

6. emplace_back 和 push_back 的区别
- push_back 传入对象。
- emplace_back 在容器内部原地构造对象，通常可减少临时对象。
- 但不是所有场景都一定更快。

==================================================
九、算法、复杂度、哈希与红黑树
==================================================

1. 常见复杂度
- 数组随机访问 O(1)
- 链表按位置访问 O(n)
- 平衡树查找/插入/删除 O(log n)
- 哈希表平均查找/插入/删除 O(1)

2. 红黑树为什么常用于 map/set
- 自平衡，能保证 O(log n)。
- 支持有序遍历和范围操作。
- 性能稳定，不像哈希表那样强依赖哈希质量。

3. unordered_map 什么时候会退化
- 哈希冲突严重时。
- 哈希函数差或恶意输入时。
- 可能退化到链表/桶遍历级别。

==================================================
十、模板、泛型编程、SFINAE、Concepts
==================================================

1. 模板是什么
- 把类型参数化，编译期生成具体代码。
- 核心价值：复用和零开销抽象。

2. 函数模板和类模板
- 函数模板通常可自动推导类型参数。
- 类模板一般需要显式指定或依赖类模板参数推导。

3. 模板特化和偏特化
- 全特化：为某个具体类型提供专门实现。
- 偏特化：针对某一类模板参数形式提供实现。
- 注意：函数模板没有偏特化，通常用重载代替。

4. 什么是 SFINAE
- Substitution Failure Is Not An Error。
- 模板替换失败不是错误，而是让编译器继续找其他候选。
- 传统现代 C++ 面试会问它如何实现类型约束。

5. Concepts 是什么
- C++20 引入的模板约束机制。
- 比 enable_if/SFINAE 更直观，错误信息更友好。
- 现在它已经是较新的面试热点，尤其在中高级岗位中。

6. Concepts 和 SFINAE 的区别
- Concepts 语义更清晰。
- 编译报错更容易理解。
- 更适合表达接口约束。

面试表达模板
- 以前模板约束常用 SFINAE 和 std::enable_if，C++20 以后更推荐 Concepts，因为代码可读性更好，也更容易让模板接口表达设计意图。

==================================================
十一、auto、decltype、constexpr、lambda
==================================================

1. auto 的作用
- 让编译器根据初始化表达式推导类型。
- 提高可读性，但不能滥用到让类型语义变模糊。

2. decltype 的作用
- 推导表达式类型。
- 常用于模板、返回值推导、泛型编程。

3. constexpr 和 const 的区别
- const 表示对象不可修改。
- constexpr 强调编译期常量或编译期可求值。
- C++14/C++17/C++20 后 constexpr 能力越来越强。

4. lambda 的本质
- 编译器生成的匿名函数对象。
- 捕获列表决定如何访问外部变量。

5. 值捕获和引用捕获
- [=] 值捕获，复制外部变量。
- [&] 引用捕获，直接引用外部变量。
- 面试追问：异步场景中引用捕获要非常谨慎，可能悬空。

==================================================
十二、异常安全
==================================================

1. C++ 异常处理关键字
- try
- catch
- throw

2. 异常安全的三个层级
- 基本保证：出错后对象仍保持有效状态，不泄漏资源。
- 强保证：操作失败时程序状态回滚，仿佛未执行。
- 不抛异常保证：承诺不会抛异常。

3. 为什么 RAII 能提高异常安全
- 栈展开时局部对象会自动析构。
- 资源可以自动释放。

4. 析构函数为什么通常不应抛异常
- 如果栈展开过程中析构再抛异常，可能导致 std::terminate。

==================================================
十三、并发编程高频题
==================================================

1. 线程创建方式
- std::thread
- std::jthread（C++20，更现代，支持协作取消）

2. 互斥相关
- mutex
- recursive_mutex
- timed_mutex
- shared_mutex

3. lock_guard 和 unique_lock 的区别
- lock_guard：轻量、简单，作用域锁。
- unique_lock：更灵活，可延迟加锁、手动解锁、配合条件变量使用。

4. 条件变量的作用
- 在线程间做等待/通知同步。
- 常配合 unique_lock 和谓词使用。

5. 为什么 wait 要配合谓词
- 防止虚假唤醒。
- 正确模式是 while/谓词检查条件。

6. 死锁的必要条件
- 互斥
- 占有且等待
- 不可剥夺
- 循环等待

7. 如何避免死锁
- 固定加锁顺序
- 一次性获取多个锁
- 使用 std::lock
- 缩小临界区
- 尽量避免嵌套锁

8. 原子操作和 mutex 的区别
- atomic 适合简单共享状态，通常更轻量。
- mutex 适合复合操作或临界区保护。
- atomic 不等于无锁万能，复杂逻辑仍可能需要锁。

9. 什么是 CAS
- Compare-And-Swap。
- 先比较内存值与预期值，相等则更新。
- 是无锁算法的基础原语之一。

10. 什么是竞态条件
- 多线程并发访问共享数据，结果依赖执行时序。
- 没有正确同步就可能出现。

==================================================
十四、C++ 内存模型与 memory_order
==================================================

1. 什么是 C++11 内存模型
- 定义了多线程程序中原子操作、可见性、重排序、同步关系等规则。
- 这是较多底层岗位会追问的重点。

2. 常见 memory_order
- memory_order_relaxed：只保证原子性，不保证顺序。
- memory_order_acquire：获取语义，阻止后续读写重排到前面。
- memory_order_release：释放语义，阻止前面读写重排到后面。
- memory_order_acq_rel：同时具有 acquire/release 语义。
- memory_order_seq_cst：最强的一致顺序保证，最容易理解，代价也通常更高。

3. acquire/release 的直观理解
- release 像“发布数据”。
- acquire 像“接收数据并确保看到发布前写入的内容”。

4. 面试怎么答内存序
- 如果业务并不需要极致优化，优先 seq_cst 保证正确性。
- 对性能敏感、同步关系明确时，再谨慎使用 acquire/release 或 relaxed。

==================================================
十五、C++17 高频补充
==================================================

1. optional
- 表示“可能有值，也可能没值”。
- 比返回裸指针或特殊值更清晰。

2. variant
- 类型安全的联合体。
- 比 union 更安全。

3. any
- 可存任意类型，但需要运行时检查和转换。

4. string_view
- 对字符串的非拥有视图。
- 优点是避免拷贝。
- 风险是生命周期问题，底层数据失效后 view 就悬空。

5. structured bindings
- 结构化绑定，方便拆解 pair、tuple、结构体。

==================================================
十六、C++20 高频新特性
==================================================

1. Concepts
- 现代模板约束核心热点。
- 建议重点准备：requires、concept、自定义约束、标准概念如 integral。

2. Ranges
- 把算法和视图组合得更自然。
- 面试追问点：它和传统 STL 算法有什么改进？
- 核心回答：可组合、懒求值、更安全、更易表达数据流。

3. Coroutines
- 近年是“中高级加分项”，不是所有岗位必问。
- 重点理解：
  - co_await
  - co_yield
  - co_return
  - promise_type
  - coroutine_handle
- 高频追问：它和线程有什么区别？
  回答：协程是用户态协作式执行流抽象，不等同于系统线程，能减少切换开销并改善异步代码结构。

4. Modules
- 用 import/export 代替部分头文件组织方式。
- 价值：降低编译依赖、减少宏污染、提升构建体验。
- 现实面试中常见追问：为什么 modules 普及比想象中慢？
  回答：工具链、构建系统、历史代码、跨平台兼容成本较高。

5. std::span
- 非拥有的连续内存视图。
- 可统一处理数组、vector、裸数组片段。
- 面试常问：它不管理内存，只是视图，要注意生命周期。

6. std::jthread
- 比 std::thread 更现代。
- 析构时会请求停止并 join，更安全。
- 配合 stop_token 可以做协作取消。

==================================================
十七、C++23 热点补充
==================================================

1. std::expected
- 表达“要么有结果，要么有错误”。
- 比异常更显式，适合很多工程场景。
- 面试追问：和 optional 的区别？
  optional 只有“有/无值”；expected 还能携带错误信息。

2. mdspan
- 多维视图抽象，不拥有数据。
- 常用于数值计算、高性能场景。

3. print / format 方向
- 现代格式化输出更类型安全、可读性更好。
- 面试不一定深问，但可能作为“你了解哪些 C++23 改进”时的答题点。

4. deducing this
- 改进成员函数模板和对象参数表达方式。
- 更偏进阶，了解即可。

5. 对 ranges 的继续增强
- C++23 继续增强 ranges/view 生态。
- 对“现代 STL 风格”岗位是一个加分点。

面试建议
- 如果岗位要求“modern C++”，至少要能说清 C++20。
- 如果是中高级岗位，最好顺带提一句 C++23 的 expected 和 mdspan。

==================================================
十八、项目与工程实践高频追问
==================================================

1. 你在项目中如何避免内存泄漏
- 优先用 RAII。
- 资源使用智能指针、容器、锁封装。
- 配合 sanitizers、valgrind、代码审查。

2. 你做过哪些性能优化
- 减少不必要拷贝，使用 move 语义。
- 预分配 reserve。
- 减少锁竞争，缩小临界区。
- 选择合适容器。
- 避免频繁堆分配。
- 关注缓存局部性。

3. 为什么 vector 往往比 list 更快
- 虽然 list 插删复杂度低，但 vector 连续内存带来更好的缓存局部性。
- 实际工程中 vector 经常更快。

4. 你如何设计线程安全队列
- 互斥锁 + 条件变量版本最常见。
- 进一步可讨论无锁队列、CAS、ABA 问题。

5. 如何定位线上崩溃
- core dump
- gdb
- 日志
- 地址消毒器 ASan
- 线程消毒器 TSan
- 最小复现

==================================================
十九、近两年更容易被问到的“现代 C++观念题”
==================================================

1. 为什么现代 C++ 强调“表达所有权”
- 代码正确性高度依赖资源生命周期。
- unique_ptr、shared_ptr、span、string_view 等类型都在表达“拥有/不拥有”关系。
- 面试里如果能主动提 ownership，会显得更工程化。

2. 为什么推荐 Rule of 0
- 把资源管理交给标准库类型。
- 减少手写特殊成员函数。
- 降低 bug 概率。

3. 什么是零开销抽象
- 不用某个抽象时，不为它付出成本。
- 使用抽象时，其性能不应劣于手写等价代码。
- 这是 C++ 很核心的设计哲学。

4. 为什么 string_view、span 很有用但也危险
- 它们是非拥有视图，避免拷贝。
- 但如果底层对象先销毁，就会悬空。

5. Concepts 为什么被认为是现代 C++ 重要升级
- 因为它让模板接口具备了“可读的契约”。
- 这在大型工程和库设计里价值很高。

==================================================
二十、面试快答题库
==================================================

1. 什么情况下会发生对象切片
- 用基类对象按值接收或赋值派生类对象时，派生类部分被切掉。

2. 什么是野指针
- 指向非法地址的指针，可能未初始化、已释放或越界。

3. 什么是悬空指针
- 指针指向的对象已经被释放，但指针没置空。

4. nullptr 和 NULL 区别
- nullptr 是 C++11 引入的空指针字面量，类型安全。
- NULL 本质常是 0 或宏，重载解析可能出问题。

5. volatile 有什么用
- 告诉编译器不要对该对象访问做某些优化。
- 它不等于线程同步工具，不能替代 atomic 或 mutex。

6. explicit 有什么用
- 防止单参数构造函数发生隐式类型转换。

7. friend 有什么用
- 允许指定函数或类访问私有成员。
- 破坏部分封装，要谨慎使用。

8. override 有什么用
- 显式说明重写基类虚函数。
- 编译器可帮助检查签名错误。

9. final 有什么用
- 禁止继续继承类，或禁止继续重写某虚函数。

10. 为什么虚函数不能是 static
- static 成员函数没有 this 指针，虚函数依赖对象实例做动态分派。

==================================================
二十二、建议重点准备的回答模板
==================================================

1. 为什么优先用 unique_ptr 而不是裸指针
- 因为它能明确表达独占所有权，自动释放资源，减少泄漏风险，语义和性能通常都比 shared_ptr 更合适。

2. 什么时候用 shared_ptr
- 当对象确实需要被多个所有者共享生命周期时才用；如果只是观察，不拥有对象，就用裸指针、引用或 weak_ptr 视场景表达。

3. 为什么 move 语义重要
- 它避免了很多昂贵的深拷贝，特别是在容器扩容、返回大对象、资源封装类型中，可以显著提升性能。

4. 为什么现代 C++ 强调 RAII
- 因为资源管理是 bug 高发区，RAII 把资源绑定到对象生命周期，既提升异常安全，也减少手工清理出错。

5. 你如何理解 Concepts
- 它是模板约束机制，让泛型代码的接口更清楚、报错更友好，是比传统 enable_if 更现代的写法。

==================================================
二十四、答题策略
==================================================

1. 先给定义，再给场景，再给风险点
例如：
- 什么是 shared_ptr？
- 它是共享所有权智能指针，通过引用计数管理对象生命周期。
- 适合多个对象共同持有资源。
- 风险是循环引用和额外计数开销。

2. 多提“语义”
- 所有权语义
- 生命周期语义
- 线程安全语义
- 异常安全语义

3. 多提“工程权衡”
- 不是只背定义，而是说明什么时候用、为什么不用、代价是什么。

4. 新标准回答不要贪多
- C++20 重点讲 Concepts、Ranges、span、jthread。
- C++23 重点讲 expected。
- Coroutines/Modules 会一点原理和适用场景即可。
